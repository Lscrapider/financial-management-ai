# Agent Instructions

每次开始处理本项目任务时，先阅读本文件，并按以下要求选择和使用 skill / tool。

## 优先级

- 当前用户明确指令优先。
- 本文件作为本项目的开发规范和执行约束。
- 如果 skill / tool 的官方使用规则与本文件冲突，优先遵循对应 skill / tool 的官方规则。
- 不确定时先说明假设，再执行。

## 写代码/改代码

- 使用 Superpowers 插件及其适用 skill：做新功能、改行为、实现功能、修 bug、调试、计划、头脑风暴、并行 agent 协作、代码审查和收尾验证时，按 Superpowers 的流程执行。
- 不强制新增单元测试和主动启动前端服务或打开浏览器查看页面；如需新增单元测试，先询问确认。可以直接运行项目已有测试、类型检查、静态检查或手动验证。
- 使用 scrapider-guidelines：写代码时遵循实现逻辑、最小改动、验证标准、Spring Boot 分层规范。
- 使用 codegraph_*：分析“这个功能怎么流转”“这个方法谁调用”“改这里影响哪里”等代码结构和调用关系问题。

## 多 Agent 协作

- 对于可以并行、相互独立的任务，可以拆分给多个 agent 分别执行。
- 对于存在强依赖关系的任务，不要盲目并行。
- Codex 主线负责整体任务拆解、方案校验、代码审核、结果整合和最终验收。
- 其他 agent 的输出必须经过 Codex 主线复核后再采用。

## 做前端设计

- 使用 impeccable：处理 UI 设计、重构、视觉打磨、交互、响应式、可访问性、信息架构。

## 执行要求

- 修改前先理解现有项目结构、技术栈、代码约定和上下文。
- 优先做最小必要改动，避免无关重构。
- 修改后进行必要的测试、类型检查、静态检查或手动验证。
- 如果无法验证，必须说明未验证的原因和潜在风险。

## 语言要求

- 项目文档、代码注释和git提交信息使用中文。
- 面向用户的说明优先使用中文。
