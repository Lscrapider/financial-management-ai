

# 标的标签计算规则

本文档用于定义 report 生成阶段中，当前标的 `currentScenes` 的 7 大类标签计算方式。

核心目标：

```text
chunk 入库时已经有 7 大类标签；
当前标的分析时，也要生成同一套 7 大类标签，并且每个标签带 0~1 的分数；
后续根据当前标的标签分数，动态召回更匹配的知识 chunk。
```

---

## 1. 通用约定

### 1.1 分数范围

所有标签分数统一归一化到 `0 ~ 1`：

```python
score = 0      # 不命中 / 无明显信号
score = 0.5    # 中等强度
score = 1.0    # 强信号
```

统一使用：

```python
def clamp(x):
    return min(max(x, 0), 1)
```

### 1.2 精度优先的归一化函数

为了提高数值准度，不建议所有指标都直接使用固定阈值线性映射。第一版可以保留 `clamp` 作为兜底，但在数据充足时，优先使用历史分位数、Robust Z-Score、Sigmoid 和时间衰减。

#### 1.2.1 Sigmoid 平滑映射

适合处理涨跌幅、成交量倍率、换手率、波动率等连续指标。

```python
import math

def sigmoid_score(x, center, scale):
    return 1 / (1 + math.exp(-(x - center) / scale))
```

含义：

```text
center：开始明显命中的中心点。
scale：分数上升速度，越小越敏感。
```

例如：

```python
price_rise = sigmoid_score(change_pct, center=2.0, scale=1.2)
```

这表示涨幅接近 2% 时开始明显命中，涨幅越大分数越接近 1。

#### 1.2.2 历史分位数评分

适合 PE、PB、成交额、换手率、波动率这类不同资产差异很大的指标。

```python
def percentile_rank(value, history_values):
    valid_values = [v for v in history_values if v is not None]
    if not valid_values:
        return None
    return sum(1 for v in valid_values if v <= value) / len(valid_values)
```

例如：

```python
high_turnover = percentile_rank(turnover_rate, turnover_rate_history_250d)
low_pb = 1 - percentile_rank(pb, pb_history_5y)
```

含义：

```text
当前换手率处于历史 90% 分位，则 high_turnover = 0.90。
当前 PB 处于历史 20% 分位，则 low_pb = 0.80。
```

如果有历史数据，优先使用历史分位数，而不是固定阈值。

#### 1.2.3 Robust Z-Score 异动评分

适合判断“是否异常”，例如成交量突然放大、新闻数量突然增加、日内波动突然变大。

```python
def robust_z_score(value, median, mad):
    return (value - median) / (1.4826 * mad + 1e-9)

def z_to_score(z, center=1.5, scale=1.0):
    return sigmoid_score(z, center=center, scale=scale)
```

例如：

```python
volume_z = robust_z_score(current_volume, median_volume_60d, mad_volume_60d)
volume_spike = z_to_score(volume_z, center=1.8, scale=0.7)
```

含义：

```text
Robust Z-Score 比普通均值比值更抗极端值影响。
适合真实行情中成交量偶尔异常巨大的情况。
```

#### 1.2.4 时间衰减权重

适合新闻、公告、社交讨论、研报等文本信号。

```python
time_weight = math.exp(-hours_since_publish / half_life_hours)
```

建议：

```text
短线报告：half_life_hours = 24 ~ 72。
中期报告：half_life_hours = 168，也就是 7 天。
```

#### 1.2.5 多证据合成公式

多个信号同时支持一个标签时，不建议永远简单平均。可以使用加权平均，也可以使用 noisy-or。

加权平均：

```python
score = sum(w_i * s_i for w_i, s_i in signals) / sum(w_i for w_i, s_i in signals)
```

noisy-or：

```python
score = 1 - product(1 - w_i * s_i for w_i, s_i in signals)
```

如果代码实现：

```python
def noisy_or(scores):
    result = 1.0
    for score in scores:
        result *= (1 - clamp(score))
    return 1 - result
```

区别：

```text
加权平均：适合多个证据都需要综合考虑的标签。
noisy-or：适合任意一个强证据出现就应该明显提高分数的标签，例如 news_driven、policy_driven、institutional_behavior、risk_control。
```

### 1.3 模块输出结构

每个模块统一输出：

```json
{
  "module": "price",
  "score": 0.74,
  "level": "high",
  "direction": "positive",
  "tags": {
    "price_rise": 0.80,
    "near_recent_high": 0.75
  },
  "evidence": [
    "当日涨幅较明显",
    "当前价格接近近期高位"
  ]
}
```

### 1.4 模块总分计算

模块总分不建议简单取平均，因为很多标签不会同时出现。

第一版推荐：

```python
module_score = max(tag_scores)
```

如果希望多个标签同时命中时提高模块重要性，可以使用：

```python
module_score = 0.7 * max(tag_scores) + 0.3 * avg(non_zero_tag_scores)
```

对于 `risk_strategy`、`news_driven`、`policy_driven` 这类多证据累积型模块，可以优先使用 noisy-or。

## 2. 基础变量计算

后续 7 个模块都会复用这些基础变量。

| 变量 | 含义 | 公式 / 计算方式 |
|---|---|---|
| `change_pct` | 当日涨跌幅百分比 | 行情接口直接获取，例如 `3.5` 表示上涨 3.5% |
| `price_rise` | 上涨强度 | 优先：`sigmoid_score(change_pct, center=2.0, scale=1.2)`；简化：`clamp(change_pct / 5)`，仅当 `change_pct > 0` 生效 |
| `price_drop` | 下跌强度 | 优先：`sigmoid_score(abs(change_pct), center=2.0, scale=1.2)`；简化：`clamp(abs(change_pct) / 5)`，仅当 `change_pct < 0` 生效 |
| `price_move` | 价格异动强度 | 优先：`sigmoid_score(abs(change_pct), center=2.0, scale=1.2)`；简化：`clamp(abs(change_pct) / 5)` |
| `volume_ratio_5d` | 当前成交量 / 近 5 日均量 | `current_volume / avg_volume_5d` |
| `volume_ratio_20d` | 当前成交量 / 近 20 日均量 | `current_volume / avg_volume_20d` |
| `volume_expand` | 放量强度 | 优先：`z_to_score(robust_z_score(current_volume, median_volume_60d, mad_volume_60d), center=1.0, scale=0.8)`；简化：`clamp((volume_ratio_5d - 1) / 2)` |
| `volume_shrink` | 缩量强度 | `clamp((1 - volume_ratio_5d) / 0.5)` |
| `volume_spike` | 爆量强度 | 优先：`z_to_score(robust_z_score(current_volume, median_volume_60d, mad_volume_60d), center=1.8, scale=0.7)`；简化：`clamp((volume_ratio_20d - 1) / 3)` |
| `high_turnover` | 高换手强度 | 优先：`percentile_rank(turnover_rate, turnover_rate_history_250d)`；简化：`clamp(turnover_rate / turnover_threshold)` |
| `low_turnover` | 低换手强度 | `1 - high_turnover` |
| `position_20d` | 当前价格在近 20 日区间的位置 | `(current_price - low_20d) / (high_20d - low_20d)` |
| `near_recent_high` | 接近近期高位 | `clamp((position_20d - 0.8) / 0.2)` |
| `near_recent_low` | 接近近期低位 | `clamp((0.2 - position_20d) / 0.2)` |
| `breakout` | 突破前高 | `1 if current_price > prev_high_20d else 0` |
| `break_recent_low` | 跌破前低 | `1 if current_price < prev_low_20d else 0` |
| `close_weak` | 收盘弱 | `clamp((intraday_high - close_price) / (intraday_high - intraday_low))` |
| `upper_shadow` | 上影线强度 | `clamp((high_price - max(open_price, close_price)) / (high_price - low_price))` |
| `large_intraday_move` | 日内大波动 | `clamp((high_price - low_price) / prev_close / 0.08)` |
| `volatility` | 波动率强度 | 优先：`percentile_rank(atr_14 / current_price, volatility_history_250d)`；简化：`clamp((atr_14 / current_price) / volatility_threshold)` |
| `support_distance` | 距离支撑位强度 | `clamp(((current_price - support_price) / current_price) / support_distance_threshold)` |

注意：

```text
turnover_threshold 需要按资产类型配置。
银行股可以低一些，普通股票居中，小盘题材股和可转债可以高一些。
如果某个指标有足够历史数据，优先使用历史分位数；数据不足时，再退回固定阈值。
```

## 3. asset：资产类型标签

### 3.1 推荐方式

`asset` 模块主要基于数据库字段和资产元数据，不建议使用 LLM。

| 标签 | 中文含义 | 推荐方式 | 计算方式 |
|---|---|---|---|
| `general` | 通用投资经验 | 规则 | 当前标的无法明确归类，或报告需要通用投资经验时设为 `1.0` |
| `stock` | 股票 | 规则 | `asset_type == "stock" -> 1.0` |
| `index` | 指数 | 规则 | `asset_type == "index" -> 1.0` |
| `convertible_bond` | 可转债 | 规则 | `asset_type == "convertible_bond" -> 1.0` |
| `fund` | 基金 | 规则 | `asset_type == "fund" or asset_type == "ETF" -> 1.0` |
| `bank_stock` | 银行股 | 规则 | `asset_type == "stock" and industry == "bank" -> 1.0` |
| `low_price_stock` | 低价股 | 规则 | `asset_type == "stock" and current_price <= low_price_threshold -> 1.0` |
| `large_cap_stock` | 大盘股 | 规则 | `market_cap_percentile >= 0.8 -> 1.0` |
| `small_cap_stock` | 小盘股 | 规则 | `market_cap_percentile <= 0.2 -> 1.0` |

### 3.2 示例

```json
{
  "module": "asset",
  "score": 1.0,
  "level": "high",
  "direction": "neutral",
  "tags": {
    "stock": 1.0,
    "bank_stock": 1.0,
    "low_price_stock": 0.8
  },
  "evidence": [
    "标的类型为股票",
    "所属行业为银行",
    "当前价格处于低价股区间"
  ]
}
```

## 4. price：价格位置标签

### 4.1 推荐方式

`price` 模块全部建议用规则计算，不建议用 LLM。

| 标签 | 中文含义 | 推荐方式 | 公式 / 计算方式 |
|---|---|---|---|
| `price_rise` | 价格上涨 | 规则 | 优先：`score = sigmoid_score(change_pct, center=2.0, scale=1.2)`；简化：`score = clamp(change_pct / 5)`，仅当 `change_pct > 0` 生效 |
| `price_drop` | 价格下跌 | 规则 | 优先：`score = sigmoid_score(abs(change_pct), center=2.0, scale=1.2)`；简化：`score = clamp(abs(change_pct) / 5)`，仅当 `change_pct < 0` 生效 |
| `sideways` | 横盘 | 规则 | `range_pct_20d = (high_20d - low_20d) / low_20d`；`score = clamp((0.08 - range_pct_20d) / 0.08)` |
| `near_recent_high` | 接近近期高位 | 规则 | `position_20d = (current_price - low_20d) / (high_20d - low_20d)`；`score = clamp((position_20d - 0.8) / 0.2)` |
| `near_recent_low` | 接近近期低位 | 规则 | `score = clamp((0.2 - position_20d) / 0.2)` |
| `breakout` | 突破 | 规则 | `score = 1.0 if current_price > prev_high_20d else 0.0` |
| `pullback` | 回调 | 规则 | `score = uptrend * clamp((recent_high - current_price) / recent_high / pullback_threshold)` |
| `gap_up` | 跳空高开 | 规则 | `score = clamp((open_price - prev_high) / prev_close / gap_threshold)`，仅当 `open_price > prev_high` 生效 |
| `gap_down` | 跳空低开 | 规则 | `score = clamp((prev_low - open_price) / prev_close / gap_threshold)`，仅当 `open_price < prev_low` 生效 |

### 4.2 说明

```text
price 模块描述“当前价格状态”。
它不负责判断后续策略，后续策略由 risk_strategy 模块组合生成。
```

## 5. volume：成交量 / 换手标签

### 5.1 推荐方式

`volume` 模块主要基于成交量、成交额、换手率、量价关系，建议用规则计算。

| 标签 | 中文含义 | 推荐方式 | 公式 / 计算方式 |
|---|---|---|---|
| `volume_expand` | 放量 | 规则 | 优先：`score = z_to_score(robust_z_score(current_volume, median_volume_60d, mad_volume_60d), center=1.0, scale=0.8)`；简化：`score = clamp((volume_ratio_5d - 1) / 2)` |
| `volume_shrink` | 缩量 | 规则 | `score = clamp((1 - volume_ratio_5d) / 0.5)` |
| `high_turnover` | 高换手 | 规则 | 优先：`score = percentile_rank(turnover_rate, turnover_rate_history_250d)`；简化：`score = clamp(turnover_rate / turnover_threshold)` |
| `low_turnover` | 低换手 | 规则 | `score = 1 - high_turnover` |
| `volume_price_confirm` | 量价配合 | 规则 | `score = max(price_rise * volume_expand, price_drop * volume_shrink)` |
| `volume_price_divergence` | 量价背离 | 规则 | `score = max(price_rise * volume_shrink, volume_expand * close_weak)` |
| `volume_spike` | 成交量突然放大 | 规则 | 优先：`score = z_to_score(robust_z_score(current_volume, median_volume_60d, mad_volume_60d), center=1.8, scale=0.7)`；简化：`score = clamp((volume_ratio_20d - 1) / 3)` |
| `volume_dry_up` | 成交枯竭 | 规则 | `score = clamp((0.5 - volume_ratio_20d) / 0.5)` |

### 5.2 阈值建议

```text
普通股票 turnover_threshold 可先设为 10%。
银行股 turnover_threshold 可先设为 3% ~ 5%。
小盘题材股 turnover_threshold 可先设为 15%。
可转债需要单独设置更高阈值。

精度优先时，优先使用该标的历史换手率分位数；只有历史数据不足时，才使用固定阈值。
```

## 6. trend：趋势结构标签

### 6.1 推荐方式

`trend` 模块主要用 K 线结构和均线关系判断，第一版建议规则计算。

| 标签 | 中文含义 | 推荐方式 | 公式 / 计算方式 |
|---|---|---|---|
| `uptrend` | 上升趋势 | 规则 | `MA5 > MA10 > MA20` 时基础分 `0.8`；若 `current_price > MA5` 加 `0.1`；若 `current_price > MA20` 加 `0.1` |
| `downtrend` | 下降趋势 | 规则 | `MA5 < MA10 < MA20` 时基础分 `0.8`；若 `current_price < MA5` 加 `0.1`；若 `current_price < MA20` 加 `0.1` |
| `range_bound` | 区间震荡 | 规则 | `range_pct_20d = (high_20d - low_20d) / low_20d`；`score = clamp((0.10 - range_pct_20d) / 0.10)` |
| `rebound` | 反弹 | 规则 | `score = downtrend * price_rise * clamp((current_price - recent_low) / recent_low / rebound_threshold)` |
| `trend_reversal` | 趋势反转 | 规则为主 | `score = max(downtrend_to_up_signal, uptrend_to_down_signal)`；例如均线开始金叉 / 死叉、价格重新站上 / 跌破 MA20 |
| `breakout_from_range` | 横盘突破 | 规则 | `score = range_bound_recent * breakout * (0.7 + 0.3 * volume_expand)` |
| `failed_breakout` | 突破失败 | 规则 | `score = breakout_recent * close_weak * upper_shadow` |

### 6.2 说明

```text
trend 看的是一段时间结构，不是单日涨跌。
例如 price_rise 高，只能说明今天涨了，不一定说明 uptrend 已经形成。
```

## 7. valuation：估值 / 基本面标签

### 7.1 推荐方式

`valuation` 模块中的 PE / PB / 股息率建议规则计算；`valuation_trap` 和 `fundamental_risk` 可以用 LLM 或本地模型辅助分析公告、财报、新闻。

| 标签 | 中文含义 | 推荐方式 | 公式 / 计算方式 |
|---|---|---|---|
| `low_pe` | 低 PE | 规则 | 优先：`score = 1 - percentile_rank(pe, pe_history_or_industry_pe_distribution)`；简化：有行业均值时 `score = clamp((industry_avg_pe - pe) / industry_avg_pe)` |
| `high_pe` | 高 PE | 规则 | 优先：`score = percentile_rank(pe, pe_history_or_industry_pe_distribution)`；简化：有行业均值时 `score = clamp((pe - industry_avg_pe) / industry_avg_pe)` |
| `low_pb` | 低 PB | 规则 | 优先：`score = 1 - percentile_rank(pb, pb_history_or_industry_pb_distribution)`；简化：`score = clamp((1 - pb) / 0.5)`，适合银行等 PB 重要行业 |
| `high_pb` | 高 PB | 规则 | 优先：`score = percentile_rank(pb, pb_history_or_industry_pb_distribution)`；简化：`score = clamp((pb - industry_avg_pb) / industry_avg_pb)` |
| `high_dividend` | 高股息 | 规则 | 优先：`score = percentile_rank(dividend_yield, dividend_yield_history_or_industry_distribution)`；简化：`score = clamp(dividend_yield / 0.06)` |
| `valuation_repair` | 估值修复 | 规则为主 | `score = low_valuation * price_rise * uptrend` |
| `valuation_trap` | 低估值陷阱 | 混合 | `score = low_valuation * fundamental_negative` |
| `fundamental_risk` | 基本面风险 | LLM / 本地模型 / 财务规则 | 从财报、公告、新闻中识别盈利下滑、现金流恶化、资产质量下降、监管处罚、坏账上升等，输出 `0~1` |

### 7.2 关键基础变量

```python
low_valuation = max(low_pe, low_pb, high_dividend)
```

```python
fundamental_negative = LLM_or_local_model(
    financial_report_summary,
    announcements,
    news
)
```

### 7.3 说明

```text
低 PE / 低 PB 不一定等于机会。
如果同时存在基本面恶化，应该提高 valuation_trap 和 fundamental_risk。
```

## 8. sentiment：情绪 / 异动标签

### 8.1 推荐方式总表

`sentiment` 模块不是只输出一个情绪值，而是输出多个标签及其分数。

| 标签 | 中文含义 | 推荐方式 | 是否需要 LLM / 模型 | 公式 / 计算方式 |
|---|---|---|---|---|
| `market_attention_rise` | 关注度上升 | 规则为主 | 不一定需要 | 优先：`attention_z = robust_z_score(current_24h_mentions, median_daily_mentions_60d, mad_daily_mentions_60d)`；`score = z_to_score(attention_z, center=1.5, scale=0.8)`；简化：`attention_ratio = current_24h_mentions / avg_daily_mentions_7d`；`score = clamp((attention_ratio - 1) / 2)` |
| `short_term_emotion` | 短线情绪升温 | 混合 | 建议用 FinBERT / 本地模型 | `score = 0.35 * sentiment_positive + 0.25 * price_rise + 0.25 * volume_expand + 0.15 * market_attention_rise` |
| `panic_selling` | 恐慌抛售 | 混合 | 建议用 FinBERT / 本地模型 | `score = 0.30 * price_drop + 0.25 * volume_expand + 0.25 * sentiment_negative + 0.20 * break_recent_low` |
| `news_driven` | 消息驱动 | 混合 | 建议用 LLM | 推荐：`score = 1 - product(1 - event_importance_i * time_weight_i * max(price_move, volume_expand) for each event_i)`；简化：`score = event_importance * max(price_move, volume_expand)` |
| `policy_driven` | 政策驱动 | 模型为主 | 建议用 LLM | 推荐：`score = 1 - product(1 - policy_relevance_i * policy_importance_i * time_weight_i for each policy_i)`；简化：`score = policy_relevance * policy_importance` |
| `sector_rotation` | 板块轮动 | 规则为主 | 不一定需要 | `score = 0.40 * sector_relative_strength + 0.30 * sector_breadth + 0.30 * sector_volume_expand` |
| `weak_sentiment` | 情绪偏弱 | 混合 | 建议用 FinBERT / 本地模型 | `score = 0.35 * sentiment_negative + 0.25 * price_drop + 0.20 * low_attention + 0.20 * rebound_weak` |
| `herding_effect` | 羊群效应 / 从众行为 | 混合 | 建议用 LLM 或关键词模型 | `score = 0.30 * market_attention_rise + 0.25 * high_turnover + 0.25 * volume_expand + 0.20 * llm_herding_signal` |
| `institutional_behavior` | 机构 / 基金行为 | 混合 | 建议用 LLM 或结构化资金数据 | 推荐：`score = 1 - product(1 - s for s in [fund_flow_signal, institutional_news_signal, analyst_report_signal, llm_institutional_signal])`；简化：`score = max(fund_flow_signal, institutional_news_signal, analyst_report_signal, llm_institutional_signal)` |

### 8.2 FinBERT 情绪聚合

FinBERT 每条文本通常输出：

```json
{"label": "positive", "score": 0.87}
```

为了让情绪分数更稳定，推荐使用“置信度 × 时间衰减 × 来源权重”的加权方式。

```python
time_weight = exp(-hours_since_publish / half_life_hours)
source_weight = get_source_weight(source_type)
final_weight = time_weight * source_weight
```

建议来源权重：

```text
公告 / 财报：1.00
主流财经新闻：0.80
研报摘要：0.70
社交媒体：0.40
普通论坛 / 评论：0.25
```

聚合公式：

```python
sentiment_positive = sum(confidence_i * final_weight_i for label_i == "positive") / sum(final_weight_i)
sentiment_negative = sum(confidence_i * final_weight_i for label_i == "negative") / sum(final_weight_i)
sentiment_neutral  = sum(confidence_i * final_weight_i for label_i == "neutral")  / sum(final_weight_i)
```

综合情绪方向：

```python
sentiment_net = sentiment_positive - sentiment_negative
sentiment_strength = abs(sentiment_net)
```

说明：

```text
sentiment_positive / sentiment_negative 是基础变量，不一定直接作为最终标签。
它们会参与 short_term_emotion、panic_selling、weak_sentiment、stop_loss_plan 等标签计算。
```

### 8.3 LLM / 本地模型输出建议

LLM 或本地模型不要直接输出最终 sentiment 分数，而是输出可组合的中间信号：

```json
{
  "event_importance": 0.75,
  "policy_relevance": 0.60,
  "policy_importance": 0.70,
  "llm_herding_signal": 0.50,
  "llm_institutional_signal": 0.65,
  "evidence": [
    "近期公告可能影响市场预期",
    "文本中出现机构调研和基金持仓相关信息"
  ]
}
```

## 9. risk_strategy：风险 / 策略标签

### 9.1 推荐方式总表

`risk_strategy` 模块主要由前面 6 个模块的标签组合生成。

LLM 可以参与解释和少数语义风险识别，但不建议完全由 LLM 直接打最终分。

| 标签 | 中文含义 | 推荐方式 | 是否需要 LLM / 模型 | 公式 / 计算方式 |
|---|---|---|---|---|
| `chase_high_risk` | 追高风险 | 规则为主 | 不需要 | `score = 0.25 * price_rise + 0.25 * near_recent_high + 0.20 * volume_expand + 0.15 * high_turnover + 0.15 * short_term_emotion` |
| `false_breakout_risk` | 假突破风险 | 规则为主 | 不需要 | `score = 0.35 * breakout + 0.25 * close_weak + 0.20 * upper_shadow + 0.20 * volume_expand` |
| `liquidity_risk` | 流动性风险 | 规则为主 | 不需要 | `score = 0.40 * low_turnover + 0.30 * low_volume + 0.30 * wide_spread` |
| `drawdown_risk` | 回撤风险 | 规则为主 | 不需要 | `score = 0.30 * near_recent_high + 0.25 * volatility + 0.25 * price_rise + 0.20 * support_distance` |
| `valuation_trap_risk` | 估值陷阱风险 | 混合 | 建议用 LLM | `score = low_valuation * fundamental_negative` |
| `overheated_risk` | 过热风险 | 规则为主 | 不需要 | `score = 0.30 * price_rise + 0.25 * volume_expand + 0.25 * high_turnover + 0.20 * short_term_emotion` |
| `risk_control` | 风险控制 | 规则为主 | 不需要 | 推荐：`score = 1 - product(1 - r for r in [chase_high_risk, false_breakout_risk, liquidity_risk, drawdown_risk, valuation_trap_risk, overheated_risk])`；简化：`score = max(chase_high_risk, false_breakout_risk, liquidity_risk, drawdown_risk, valuation_trap_risk, overheated_risk)` |
| `position_control` | 仓位控制 | 规则为主 | 不需要 | `score = 0.50 * risk_control + 0.25 * volatility + 0.25 * uncertainty` |
| `wait_confirm` | 等待确认 | 混合 | 可用 LLM 辅助解释 | 推荐：`score = 1 - product(1 - s for s in [breakout * (1 - volume_expand), short_term_emotion * (1 - trend_confirmed), policy_driven * (1 - price_confirmed), sentiment_conflict])`；简化：`score = max(breakout * (1 - volume_expand), short_term_emotion * (1 - trend_confirmed), policy_driven * (1 - price_confirmed), sentiment_conflict)` |
| `observe_next_day` | 观察次日表现 | 规则为主 | 不需要 | `score = max(news_driven, policy_driven, market_attention_rise, volume_spike, large_intraday_move) * (1 - trend_confirmed)` |
| `avoid_emotional_trade` | 避免情绪交易 | 规则为主 | 不需要 | 推荐：`score = 1 - product(1 - s for s in [short_term_emotion, panic_selling, herding_effect, overheated_risk])`；简化：`score = max(short_term_emotion, panic_selling, herding_effect, overheated_risk)` |
| `take_profit_plan` | 止盈计划 | 规则为主 | 不需要 | `score = 0.30 * price_rise + 0.25 * near_recent_high + 0.25 * overheated_risk + 0.20 * drawdown_risk` |
| `stop_loss_plan` | 止损计划 | 规则为主 | 不需要 | `score = 0.35 * break_recent_low + 0.25 * downtrend + 0.20 * sentiment_negative + 0.20 * drawdown_risk` |

### 9.2 risk_strategy 关键变量

| 变量 | 含义 | 公式 / 来源 |
|---|---|---|
| `low_volume` | 低成交 | `clamp((1 - volume_ratio_5d) / 0.5)` |
| `wide_spread` | 买卖价差大 | `clamp(spread_pct / spread_threshold)` |
| `low_valuation` | 低估值强度 | `max(low_pe, low_pb, high_dividend)` |
| `fundamental_negative` | 基本面负面强度 | LLM / 本地模型 / 财务规则输出 |
| `trend_confirmed` | 趋势确认度 | 例如 `MA5 > MA10 > MA20` 且 `current_price > MA5` 时较高 |
| `price_confirmed` | 价格确认度 | 突破后是否站稳关键位 |
| `sentiment_conflict` | 情绪冲突 | 可由规则或 LLM 判断，例如情绪偏正但价格下跌、消息利好但成交不跟随 |
| `uncertainty` | 不确定性 | `max(wait_confirm, sentiment_conflict, policy_uncertainty, news_uncertainty)` |

### 9.3 risk_strategy 说明

```text
risk_strategy 不是独立原始数据模块，而是综合模块。
它应该读取 asset、price、volume、trend、valuation、sentiment 的输出，再生成风险和策略标签。
```

## 10. 哪些标签需要 LLM / 本地模型

### 10.1 强烈建议用 LLM 或本地模型

| 标签 | 原因 |
|---|---|
| `policy_driven` | 政策对不同资产、行业影响不同，不能只靠关键词 |
| `news_driven` | 需要理解事件重要性，以及行情是否可能由事件触发 |
| `institutional_behavior` | 机构行为常隐藏在研报、公告、基金持仓、调研信息里 |
| `valuation_trap_risk` | 低估值是否是陷阱，需要理解财报、公告、新闻中的基本面风险 |
| `fundamental_risk` | 需要识别盈利下滑、现金流恶化、资产质量问题、监管处罚等语义信息 |
| `herding_effect` | 文本里的从众、抱团、抢筹、净买入、净卖出等语义适合模型识别 |

### 10.2 建议用 FinBERT / 本地情绪模型

| 标签 / 基础变量 | 原因 |
|---|---|
| `sentiment_positive` | 基础正面情绪变量，供 `short_term_emotion` 等公式使用 |
| `sentiment_negative` | 基础负面情绪变量，供 `panic_selling`、`weak_sentiment` 等公式使用 |
| `sentiment_neutral` | 基础中性情绪变量，用于判断消息是否缺少方向性 |
| `short_term_emotion` | 需要文本情绪和短线行情共同判断 |
| `panic_selling` | 需要识别恐慌、抛售、负面情绪文本 |
| `weak_sentiment` | 需要识别负面情绪和市场冷淡 |

### 10.3 规则更稳定，不建议用 LLM

## 11. 精度优先的落地建议

如果目标是“数值准度优先”，推荐按下面优先级实现：

```text
第一优先级：历史分位数 percentile_rank
第二优先级：Robust Z-Score + Sigmoid
第三优先级：固定阈值 clamp
```

原因：

```text
固定阈值容易受行业和资产类型影响。
例如银行股 3% 换手率可能已经很高，但小盘题材股 3% 并不高。
历史分位数可以自动适配不同标的自身特征。
```

推荐实现方式：

```python
# 1. 有该标的历史数据时，优先用该标的历史分位
score = percentile_rank(current_value, asset_history_values)

# 2. 没有足够历史数据时，用行业分布
score = percentile_rank(current_value, industry_values)

# 3. 历史和行业数据都不足时，再退回固定阈值
score = clamp(current_value / threshold)
```

数据不足时：

```text
如果某个公式依赖的数据缺失，不要让 LLM 猜数值。
该标签可以不生成，或者生成 score = 0，并在 evidence 中记录数据缺失原因。
```

最终建议：

```text
规则负责数值稳定性；
FinBERT / 本地模型负责文本情绪；
LLM 负责政策、事件、机构行为、基本面风险等语义信号；
所有模型输出都尽量转成中间变量，再由公式合成最终标签分数。
```
