

# 标的标签计算规则

本文档用于定义 report 生成阶段中，当前标的 `currentScenes` 的 7 大类标签计算方式。

核心目标：

```text
chunk 入库时已经有 7 大类标签；
当前标的分析时，也要生成同一套 7 大类标签，并且每个标签带 0~1 的分数；
后续根据当前标的标签分数，动态召回更匹配的知识 chunk。
```

---

## 1. 通用约定

### 1.1 分数范围

所有标签分数统一归一化到 `0 ~ 1`：

```python
score = 0      # 不命中 / 无明显信号
score = 0.5    # 中等强度
score = 1.0    # 强信号
```

统一使用：

```python
def clamp(x):
    return min(max(x, 0), 1)
```

### 1.2 精度优先的归一化函数

为了提高数值准度，不建议使用固定阈值线性映射作为标签兜底。`clamp` 只用于最终边界保护，核心评分优先使用历史分位数、Robust Z-Score、Sigmoid 和时间衰减。

#### 1.2.1 Sigmoid 平滑映射

适合处理涨跌幅、成交量倍率、换手率、波动率等连续指标。

```python
import math

def sigmoid_score(x, center, scale):
    return 1 / (1 + math.exp(-(x - center) / scale))
```

含义：

```text
center：开始明显命中的中心点。
scale：分数上升速度，越小越敏感。
```

例如：

```python
price_rise = sigmoid_score(change_pct, center=2.0, scale=1.2)
```

这表示涨幅接近 2% 时开始明显命中，涨幅越大分数越接近 1。

#### 1.2.2 历史分位数评分

适合 PE、PB、成交额、换手率、波动率这类不同资产差异很大的指标。

```python
def percentile_rank(value, history_values):
    valid_values = [v for v in history_values if v is not None]
    if not valid_values:
        return None
    return sum(1 for v in valid_values if v <= value) / len(valid_values)
```

例如：

```python
high_turnover = percentile_rank(turnover_rate, turnover_rate_history_250d)
low_pb = 1 - percentile_rank(pb, pb_history_5y)
```

含义：

```text
当前换手率处于历史 90% 分位，则 high_turnover = 0.90。
当前 PB 处于历史 20% 分位，则 low_pb = 0.80。
```

历史分位数类指标不再使用固定阈值兜底。数据来源按标的历史分布、行业分布、同资产类型分布逐级降级；仍然不足时，该标签不生成或生成 `score = 0`，并在 `evidence` 中记录数据缺失原因。

#### 1.2.3 Robust Z-Score 异动评分

适合判断“是否异常”，例如成交量突然放大、新闻数量突然增加、日内波动突然变大。

```python
def robust_z_score(value, median, mad):
    return (value - median) / (1.4826 * mad + 1e-9)

def z_to_score(z, center=1.5, scale=1.0):
    return sigmoid_score(z, center=center, scale=scale)
```

例如：

```python
volume_z = robust_z_score(current_volume, median_volume_60d, mad_volume_60d)
volume_spike = z_to_score(volume_z, center=1.8, scale=0.7)
```

含义：

```text
Robust Z-Score 比普通均值比值更抗极端值影响。
适合真实行情中成交量偶尔异常巨大的情况。
```

#### 1.2.4 时间衰减权重

适合新闻、公告、社交讨论、研报等文本信号。

```python
time_weight = math.exp(-hours_since_publish / half_life_hours)
```

建议：

```text
短线报告：half_life_hours = 24 ~ 72。
中期报告：half_life_hours = 168，也就是 7 天。
```

#### 1.2.5 多证据合成公式

多个信号同时支持一个标签时，不建议永远简单平均。可以使用加权平均，也可以使用 noisy-or。

加权平均：

```python
score = sum(w_i * s_i for w_i, s_i in signals) / sum(w_i for w_i, s_i in signals)
```

noisy-or：

```python
score = 1 - product(1 - w_i * s_i for w_i, s_i in signals)
```

如果代码实现：

```python
def noisy_or(scores):
    result = 1.0
    for score in scores:
        result *= (1 - clamp(score))
    return 1 - result
```

加权求和：

```python
def weighted_sum(signals, weights):
    valid_items = [
        (signal_name, score)
        for signal_name, score in signals.items()
        if score is not None and signal_name in weights
    ]
    weight_sum = sum(weights[signal_name] for signal_name, score in valid_items)
    if weight_sum <= 0:
        return 0
    return clamp(
        sum(weights[signal_name] * score for signal_name, score in valid_items) / weight_sum
    )
```

区别：

```text
加权平均：适合多个证据都需要综合考虑的标签。
noisy-or：适合任意一个强证据出现就应该明显提高分数的标签，例如 news_driven、policy_driven、institutional_behavior、risk_control。
```

### 1.3 模块输出结构

每个模块统一输出：

```json
{
  "module": "price",
  "score": 0.74,
  "level": "high",
  "direction": "positive",
  "tags": {
    "price_rise": 0.80,
    "near_recent_high": 0.75
  },
  "evidence": [
    "当日涨幅较明显",
    "当前价格接近近期高位"
  ]
}
```

### 1.4 模块总分计算

模块总分不建议简单取平均，因为很多标签不会同时出现。

第一版推荐：

```python
module_score = max(tag_scores)
```

如果希望多个标签同时命中时提高模块重要性，可以使用：

```python
module_score = 0.7 * max(tag_scores) + 0.3 * avg(non_zero_tag_scores)
```

对于 `risk_strategy` 这类多证据累积型模块，以及 `news_driven`、`policy_driven` 这类多证据累积型标签，可以优先使用 noisy-or。

### 1.5 可配置参数约定

标签计算应拆成两层：

```text
计算公式：保持稳定，由系统内置；
公式参数：允许按默认配置、资产类型配置、用户配置逐层覆盖。
```

也就是说，页面可以让用户调整阈值、权重和敏感度，但不建议让普通用户直接修改公式本身。这样既能保持标签含义稳定，也能支持不同资产类型和不同用户风格。

不是所有公式常量都需要配置化。只有影响用户风格、资产适配、信息源信任度和模型敏感度的参数建议进入配置；标签定义型常量应作为系统内置规则固定，避免同一个标签在不同用户之间语义漂移。

#### 1.5.1 参数覆盖优先级

推荐覆盖顺序：

```text
用户自定义参数 > 资产类型参数 > 系统默认参数
```

示例：

```text
普通股票高换手优先使用该标的历史换手率分布；
银行股可以覆盖为银行行业换手率分布；
某个用户如果更信任行业分布，可以在页面把 `turnover_distribution_source` 调整为 `industry_first`。
```

如果某个参数没有用户配置，使用资产类型配置；资产类型也没有配置时，使用系统默认值。

#### 1.5.2 建议抽取的参数类型

| 参数类型 | 示例 | 用途 |
|---|---|---|
| 阈值参数 | `low_price_threshold`、`gap_threshold`、`pullback_threshold`、`rebound_threshold`、`support_distance_threshold`、`spread_threshold` | 控制什么程度算命中 |
| Sigmoid 参数 | `price_rise_center`、`price_rise_scale`、`volume_expand_center`、`volume_expand_scale`、`volume_spike_center`、`volume_spike_scale` | 控制连续指标的敏感度 |
| 权重参数 | `short_term_emotion_weights`、`panic_selling_weights`、`chase_high_risk_weights`、`overheated_risk_weights` | 控制多个信号合成一个标签时的侧重点 |
| 时间衰减参数 | `news_half_life_hours`、`policy_half_life_hours`、`research_half_life_hours` | 控制文本信号随时间衰减的速度 |
| 来源权重参数 | `announcement_source_weight`、`mainstream_news_source_weight`、`research_source_weight`、`social_media_source_weight`、`forum_source_weight` | 控制不同信息源可信度 |

#### 1.5.3 页面配置建议

页面不建议一次暴露全部参数。第一版可以分为两层：

```text
普通配置：选择参数模板；
高级配置：调整具体阈值、权重、敏感度和来源权重。
```

参数模板可以先提供：

```text
稳健型：降低短线情绪和追涨信号权重，提高风险控制权重；
平衡型：使用系统默认参数；
短线敏感型：提高涨跌幅、放量、关注度和消息驱动的敏感度；
低估值偏好：提高低 PE、低 PB、高股息相关标签权重；
风险优先：提高回撤、流动性、假突破、估值陷阱相关标签权重；
银行股专用：优先使用银行行业换手率、PB、股息率分布，提高资产质量相关权重；
可转债专用：单独配置换手率、溢价率、剩余规模、强赎相关参数。
```

#### 1.5.4 参数快照

每次生成 `currentScenes` 时，建议保存本次实际使用的参数快照。

```json
{
  "profile": "balanced",
  "overrides": {
    "asset_type": "bank_stock",
    "turnover_distribution_source": "asset_history_then_industry",
    "price_rise_center": 2.0,
    "price_rise_scale": 1.2
  }
}
```

参数快照用于后续解释、复盘和排查：同一个标的在不同用户参数下，标签分数可能不同。

#### 1.5.5 权重参数使用方式

权重参数必须在公式中显式引用，不应只出现在配置表中。

推荐使用具名权重，避免只写 `0.25,0.25,0.20` 导致顺序不清楚。

```json
{
  "chase_high_risk_weights": {
    "price_rise": 0.25,
    "near_recent_high": 0.25,
    "volume_expand": 0.20,
    "high_turnover": 0.15,
    "short_term_emotion": 0.15
  }
}
```

公式统一写成：

```python
score = weighted_sum(
    signals={
        "price_rise": price_rise,
        "near_recent_high": near_recent_high,
        "volume_expand": volume_expand,
        "high_turnover": high_turnover,
        "short_term_emotion": short_term_emotion
    },
    weights=chase_high_risk_weights
)
```

如果用户只调整其中一个权重，系统应在保存配置时重新归一化，保证有效权重之和为 `1.0`。

#### 1.5.6 分布数据来源约定

`distribution_source` 用于说明分位数评分的数据来源顺序。

例如：

```text
turnover_distribution_source = asset_history_then_industry
```

表示计算 `high_turnover` 时，先使用当前标的自身历史换手率分布；如果数据不足，再使用当前标的所属行业的换手率分布。

第一版建议支持这些取值：

| 取值 | 含义 |
|---|---|
| `asset_history` | 只使用当前标的自身历史分布 |
| `industry` | 只使用当前行业分布 |
| `asset_type` | 只使用同资产类型分布，例如股票、指数、可转债、基金 |
| `asset_history_then_industry` | 先用标的历史分布，数据不足时用行业分布 |
| `asset_history_then_asset_type` | 先用标的历史分布，数据不足时用同资产类型分布 |
| `asset_history_then_industry_then_asset_type` | 先用标的历史分布，再用行业分布，最后用同资产类型分布 |
| `industry_first` | 先用行业分布，再用标的历史分布，适合用户更信任行业横向比较的场景 |

数据不足判断建议：

```text
历史分布样本数少于 min_distribution_sample_size 时，认为数据不足。
```

`min_distribution_sample_size` 可作为系统级配置，第一版可以设为 `60`，不需要暴露到普通用户页面。

如果所有候选分布都不足：

```text
不使用固定阈值硬算；
该标签不生成，或生成 score = 0；
evidence 中记录“分布数据不足”。
```

## 2. 基础变量计算

后续 7 个模块都会复用这些基础变量。

| 变量 | 含义 | 公式 / 计算方式 |
|---|---|---|
| `change_pct` | 当日涨跌幅百分比 | 行情接口直接获取，例如 `3.5` 表示上涨 3.5% |
| `price_rise` | 上涨强度 | `sigmoid_score(change_pct, center=price_rise_center, scale=price_rise_scale)`，仅当 `change_pct > 0` 生效 |
| `price_drop` | 下跌强度 | `sigmoid_score(abs(change_pct), center=price_drop_center, scale=price_drop_scale)`，仅当 `change_pct < 0` 生效 |
| `price_move` | 价格异动强度 | `sigmoid_score(abs(change_pct), center=price_move_center, scale=price_move_scale)` |
| `volume_ratio_5d` | 当前成交量 / 近 5 日均量 | `current_volume / avg_volume_5d` |
| `volume_ratio_20d` | 当前成交量 / 近 20 日均量 | `current_volume / avg_volume_20d` |
| `volume_expand` | 放量强度 | `z_to_score(robust_z_score(current_volume, median_volume_60d, mad_volume_60d), center=volume_expand_center, scale=volume_expand_scale)` |
| `volume_shrink` | 缩量强度 | `1 - percentile_rank(current_volume, volume_history_60d)` |
| `volume_spike` | 爆量强度 | `z_to_score(robust_z_score(current_volume, median_volume_60d, mad_volume_60d), center=volume_spike_center, scale=volume_spike_scale)` |
| `high_turnover` | 高换手强度 | `percentile_rank(turnover_rate, turnover_rate_distribution)` |
| `low_turnover` | 低换手强度 | `1 - high_turnover` |
| `position_20d` | 当前价格在近 20 日区间的位置 | `(current_price - low_20d) / (high_20d - low_20d)` |
| `near_recent_high` | 接近近期高位 | `clamp((position_20d - 0.8) / 0.2)` |
| `near_recent_low` | 接近近期低位 | `clamp((0.2 - position_20d) / 0.2)` |
| `breakout` | 突破前高 | `1 if current_price > prev_high_20d else 0` |
| `break_recent_low` | 跌破前低 | `1 if current_price < prev_low_20d else 0` |
| `close_weak` | 收盘弱 | `clamp((intraday_high - close_price) / (intraday_high - intraday_low))` |
| `upper_shadow` | 上影线强度 | `clamp((high_price - max(open_price, close_price)) / (high_price - low_price))` |
| `large_intraday_move` | 日内大波动 | `clamp((high_price - low_price) / prev_close / 0.08)` |
| `volatility` | 波动率强度 | `percentile_rank(atr_14 / current_price, volatility_history_250d)` |
| `support_distance` | 距离支撑位强度 | `clamp(((current_price - support_price) / current_price) / support_distance_threshold)` |

注意：

```text
换手率、波动率、估值、成交量这类跨资产差异很大的指标，优先使用标的自身历史分布。
如果标的历史数据不足，再使用行业分布或同资产类型分布。
如果分布数据仍不足，不使用固定阈值硬算，直接记录数据缺失。
```

## 3. asset：资产类型标签

### 3.1 推荐方式

`asset` 模块主要基于数据库字段和资产元数据，不建议使用 LLM。

| 标签 | 中文含义 | 推荐方式 | 计算方式 |
|---|---|---|---|
| `general` | 通用投资经验 | 规则 | 当前标的无法明确归类，或报告需要通用投资经验时设为 `1.0` |
| `stock` | 股票 | 规则 | `asset_type == "stock" -> 1.0` |
| `index` | 指数 | 规则 | `asset_type == "index" -> 1.0` |
| `convertible_bond` | 可转债 | 规则 | `asset_type == "convertible_bond" -> 1.0` |
| `fund` | 基金 | 规则 | `asset_type == "fund" or asset_type == "ETF" -> 1.0` |
| `bank_stock` | 银行股 | 规则 | `asset_type == "stock" and industry == "bank" -> 1.0` |
| `low_price_stock` | 低价股 | 规则 | `asset_type == "stock" and current_price <= low_price_threshold -> 1.0` |
| `large_cap_stock` | 大盘股 | 规则 | `market_cap_percentile >= 0.8 -> 1.0` |
| `small_cap_stock` | 小盘股 | 规则 | `market_cap_percentile <= 0.2 -> 1.0` |

### 3.2 示例

```json
{
  "module": "asset",
  "score": 1.0,
  "level": "high",
  "direction": "neutral",
  "tags": {
    "stock": 1.0,
    "bank_stock": 1.0,
    "low_price_stock": 0.8
  },
  "evidence": [
    "标的类型为股票",
    "所属行业为银行",
    "当前价格处于低价股区间"
  ]
}
```

### 3.3 建议配置项

| 参数 | 默认建议 | 说明 |
|---|---|---|
| `low_price_threshold` | 按市场配置 | 判断低价股的价格阈值，不同市场不应共用一个固定值 |
| `asset_type_aliases` | 系统维护 | 资产类型映射，例如 `ETF` 是否归入 `fund` |
| `industry_aliases` | 系统维护 | 行业映射，例如银行行业字段的不同来源名称归一化 |

## 4. price：价格位置标签

### 4.1 推荐方式

`price` 模块全部建议用规则计算，不建议用 LLM。

| 标签 | 中文含义 | 推荐方式 | 公式 / 计算方式 |
|---|---|---|---|
| `price_rise` | 价格上涨 | 规则 | `score = sigmoid_score(change_pct, center=price_rise_center, scale=price_rise_scale)`，仅当 `change_pct > 0` 生效 |
| `price_drop` | 价格下跌 | 规则 | `score = sigmoid_score(abs(change_pct), center=price_drop_center, scale=price_drop_scale)`，仅当 `change_pct < 0` 生效 |
| `sideways` | 横盘 | 规则 | `range_pct_20d = (high_20d - low_20d) / low_20d`；`score = clamp((0.08 - range_pct_20d) / 0.08)` |
| `near_recent_high` | 接近近期高位 | 规则 | `position_20d = (current_price - low_20d) / (high_20d - low_20d)`；`score = clamp((position_20d - 0.8) / 0.2)` |
| `near_recent_low` | 接近近期低位 | 规则 | `score = clamp((0.2 - position_20d) / 0.2)` |
| `breakout` | 突破 | 规则 | `score = 1.0 if current_price > prev_high_20d else 0.0` |
| `pullback` | 回调 | 规则 | `score = uptrend * clamp((recent_high - current_price) / recent_high / pullback_threshold)` |
| `gap_up` | 跳空高开 | 规则 | `score = clamp((open_price - prev_high) / prev_close / gap_threshold)`，仅当 `open_price > prev_high` 生效 |
| `gap_down` | 跳空低开 | 规则 | `score = clamp((prev_low - open_price) / prev_close / gap_threshold)`，仅当 `open_price < prev_low` 生效 |

### 4.2 说明

```text
price 模块描述“当前价格状态”。
它不负责判断后续策略，后续策略由 risk_strategy 模块组合生成。
```

### 4.3 建议配置项

| 参数 | 默认建议 | 说明 |
|---|---|---|
| `price_rise_center` | `2.0` | 涨幅接近多少时开始明显命中 `price_rise` |
| `price_rise_scale` | `1.2` | `price_rise` 分数上升速度 |
| `price_drop_center` | `2.0` | 跌幅接近多少时开始明显命中 `price_drop` |
| `price_drop_scale` | `1.2` | `price_drop` 分数上升速度 |
| `price_move_center` | `2.0` | 涨跌异动接近多少时开始明显命中 `price_move` |
| `price_move_scale` | `1.2` | `price_move` 分数上升速度 |
| `pullback_threshold` | 按资产类型配置 | 回调幅度阈值 |
| `gap_threshold` | 按资产类型配置 | 跳空幅度阈值 |

## 5. volume：成交量 / 换手标签

### 5.1 推荐方式

`volume` 模块主要基于成交量、成交额、换手率、量价关系，建议用规则计算。

| 标签 | 中文含义 | 推荐方式 | 公式 / 计算方式 |
|---|---|---|---|
| `volume_expand` | 放量 | 规则 | `score = z_to_score(robust_z_score(current_volume, median_volume_60d, mad_volume_60d), center=volume_expand_center, scale=volume_expand_scale)` |
| `volume_shrink` | 缩量 | 规则 | `score = 1 - percentile_rank(current_volume, volume_history_60d)` |
| `high_turnover` | 高换手 | 规则 | `score = percentile_rank(turnover_rate, turnover_rate_distribution)` |
| `low_turnover` | 低换手 | 规则 | `score = 1 - high_turnover` |
| `volume_price_confirm` | 量价配合 | 规则 | `score = max(price_rise * volume_expand, price_drop * volume_shrink)` |
| `volume_price_divergence` | 量价背离 | 规则 | `score = max(price_rise * volume_shrink, volume_expand * close_weak)` |
| `volume_spike` | 成交量突然放大 | 规则 | `score = z_to_score(robust_z_score(current_volume, median_volume_60d, mad_volume_60d), center=volume_spike_center, scale=volume_spike_scale)` |
| `volume_dry_up` | 成交枯竭 | 规则 | `score = 1 - percentile_rank(volume_ratio_20d, volume_ratio_20d_history)` |

### 5.2 阈值建议

```text
换手率不再使用固定阈值判断。
普通股票、银行股、小盘题材股和可转债都应优先使用各自的历史分布、行业分布或同资产类型分布。
这样可以避免银行股和小盘题材股共用同一个换手率阈值导致分数失真。
```

### 5.3 建议配置项

| 参数 | 默认建议 | 说明 |
|---|---|---|
| `volume_expand_center` | `1.0` | 放量 Robust Z-Score 的明显命中中心 |
| `volume_expand_scale` | `0.8` | 放量分数上升速度 |
| `volume_spike_center` | `1.8` | 爆量 Robust Z-Score 的明显命中中心 |
| `volume_spike_scale` | `0.7` | 爆量分数上升速度 |
| `volume_distribution_source` | `asset_history_then_industry` | 成交量分位数的数据来源 |
| `turnover_distribution_source` | `asset_history_then_industry` | 换手率分位数的数据来源 |

## 6. trend：趋势结构标签

### 6.1 推荐方式

`trend` 模块主要用 K 线结构和均线关系判断，第一版建议规则计算。

| 标签 | 中文含义 | 推荐方式 | 公式 / 计算方式 |
|---|---|---|---|
| `uptrend` | 上升趋势 | 规则 | `MA5 > MA10 > MA20` 时基础分 `0.8`；若 `current_price > MA5` 加 `0.1`；若 `current_price > MA20` 加 `0.1` |
| `downtrend` | 下降趋势 | 规则 | `MA5 < MA10 < MA20` 时基础分 `0.8`；若 `current_price < MA5` 加 `0.1`；若 `current_price < MA20` 加 `0.1` |
| `range_bound` | 区间震荡 | 规则 | `range_pct_20d = (high_20d - low_20d) / low_20d`；`score = clamp((0.10 - range_pct_20d) / 0.10)` |
| `rebound` | 反弹 | 规则 | `score = downtrend * price_rise * clamp((current_price - recent_low) / recent_low / rebound_threshold)` |
| `trend_reversal` | 趋势反转 | 规则为主 | `score = max(downtrend_to_up_signal, uptrend_to_down_signal)`；例如均线开始金叉 / 死叉、价格重新站上 / 跌破 MA20 |
| `breakout_from_range` | 横盘突破 | 规则 | `confirm_score = weighted_sum({"base_confirm": 1.0, "volume_expand": volume_expand}, breakout_from_range_confirm_weights)`；`score = range_bound_recent * breakout * confirm_score` |
| `failed_breakout` | 突破失败 | 规则 | `score = breakout_recent * close_weak * upper_shadow` |

### 6.2 说明

```text
trend 看的是一段时间结构，不是单日涨跌。
例如 price_rise 高，只能说明今天涨了，不一定说明 uptrend 已经形成。
```

### 6.3 建议配置项

| 参数 | 默认建议 | 说明 |
|---|---|---|
| `rebound_threshold` | 按资产类型配置 | 反弹幅度阈值 |
| `breakout_from_range_confirm_weights` | `{base_confirm:0.70, volume_expand:0.30}` | 横盘突破中基础确认和成交量确认的权重 |

## 7. valuation：估值 / 基本面标签

### 7.1 推荐方式

`valuation` 模块中的 PE / PB / 股息率建议规则计算；`valuation_trap` 和 `fundamental_risk` 可以用 LLM 或本地模型辅助分析公告、财报、新闻。

| 标签 | 中文含义 | 推荐方式 | 公式 / 计算方式 |
|---|---|---|---|
| `low_pe` | 低 PE | 规则 | `score = 1 - percentile_rank(pe, pe_distribution)` |
| `high_pe` | 高 PE | 规则 | `score = percentile_rank(pe, pe_distribution)` |
| `low_pb` | 低 PB | 规则 | `score = 1 - percentile_rank(pb, pb_distribution)` |
| `high_pb` | 高 PB | 规则 | `score = percentile_rank(pb, pb_distribution)` |
| `high_dividend` | 高股息 | 规则 | `score = percentile_rank(dividend_yield, dividend_yield_distribution)` |
| `valuation_repair` | 估值修复 | 规则为主 | `score = low_valuation * price_rise * uptrend` |
| `valuation_trap` | 低估值陷阱 | 混合 | `score = low_valuation * fundamental_negative` |
| `fundamental_risk` | 基本面风险 | LLM / 本地模型 / 财务规则 | 从财报、公告、新闻中识别盈利下滑、现金流恶化、资产质量下降、监管处罚、坏账上升等，输出 `0~1` |

### 7.2 关键基础变量

```python
low_valuation = max(low_pe, low_pb, high_dividend)
```

```python
fundamental_negative = LLM_or_local_model(
    financial_report_summary,
    announcements,
    news
)
```

如果不同来源分别输出负面强度，可以用来源权重合成：

```python
fundamental_negative = weighted_sum(
    signals={
        "financial_report_negative": financial_report_negative,
        "announcement_negative": announcement_negative,
        "news_negative": news_negative
    },
    weights=fundamental_negative_source_weights
)
```

### 7.3 说明

```text
低 PE / 低 PB 不一定等于机会。
如果同时存在基本面恶化，应该提高 valuation_trap 和 fundamental_risk。
```

### 7.4 建议配置项

| 参数 | 默认建议 | 说明 |
|---|---|---|
| `pe_distribution_source` | `asset_history_then_industry` | PE 分位数优先用标的历史，其次行业分布，再次同资产类型分布 |
| `pb_distribution_source` | `asset_history_then_industry` | PB 分位数优先用标的历史，其次行业分布，再次同资产类型分布 |
| `dividend_distribution_source` | `asset_history_then_industry` | 股息率分位数优先用标的历史，其次行业分布，再次同资产类型分布 |
| `fundamental_negative_source_weights` | `{financial_report_negative:0.45, announcement_negative:0.35, news_negative:0.20}` | 基本面负面强度的具名来源权重 |

## 8. sentiment：情绪 / 异动标签

### 8.1 推荐方式总表

`sentiment` 模块不是只输出一个情绪值，而是输出多个标签及其分数。

| 标签 | 中文含义 | 推荐方式 | 是否需要 LLM / 模型 | 公式 / 计算方式 |
|---|---|---|---|---|
| `market_attention_rise` | 关注度上升 | 规则为主 | 不一定需要 | `attention_z = robust_z_score(current_24h_mentions, median_daily_mentions_60d, mad_daily_mentions_60d)`；`score = z_to_score(attention_z, center=attention_center, scale=attention_scale)` |
| `short_term_emotion` | 短线情绪升温 | 混合 | 建议用 FinBERT / 本地模型 | `score = weighted_sum({"sentiment_positive": sentiment_positive, "price_rise": price_rise, "volume_expand": volume_expand, "market_attention_rise": market_attention_rise}, short_term_emotion_weights)` |
| `panic_selling` | 恐慌抛售 | 混合 | 建议用 FinBERT / 本地模型 | `score = weighted_sum({"price_drop": price_drop, "volume_expand": volume_expand, "sentiment_negative": sentiment_negative, "break_recent_low": break_recent_low}, panic_selling_weights)` |
| `news_driven` | 消息驱动 | 混合 | 建议用 LLM | `score = 1 - product(1 - event_importance_i * time_weight_i * max(price_move, volume_expand) for each event_i)` |
| `policy_driven` | 政策驱动 | 模型为主 | 建议用 LLM | `score = 1 - product(1 - policy_relevance_i * policy_importance_i * time_weight_i for each policy_i)` |
| `sector_rotation` | 板块轮动 | 规则为主 | 不一定需要 | `score = weighted_sum({"sector_relative_strength": sector_relative_strength, "sector_breadth": sector_breadth, "sector_volume_expand": sector_volume_expand}, sector_rotation_weights)` |
| `weak_sentiment` | 情绪偏弱 | 混合 | 建议用 FinBERT / 本地模型 | `score = weighted_sum({"sentiment_negative": sentiment_negative, "price_drop": price_drop, "low_attention": low_attention, "rebound_weak": rebound_weak}, weak_sentiment_weights)` |
| `herding_effect` | 羊群效应 / 从众行为 | 混合 | 建议用 LLM 或关键词模型 | `score = weighted_sum({"market_attention_rise": market_attention_rise, "high_turnover": high_turnover, "volume_expand": volume_expand, "llm_herding_signal": llm_herding_signal}, herding_effect_weights)` |
| `institutional_behavior` | 机构 / 基金行为 | 混合 | 建议用 LLM 或结构化资金数据 | `score = 1 - product(1 - s for s in [fund_flow_signal, institutional_news_signal, analyst_report_signal, llm_institutional_signal])` |

### 8.2 FinBERT 情绪聚合

FinBERT 每条文本通常输出：

```json
{"label": "positive", "score": 0.87}
```

为了让情绪分数更稳定，推荐使用“置信度 × 时间衰减 × 来源权重”的加权方式。

```python
time_weight = exp(-hours_since_publish / half_life_hours)
source_weight = get_source_weight(source_type)
final_weight = time_weight * source_weight
```

建议来源权重：

```text
公告 / 财报：1.00
主流财经新闻：0.80
研报摘要：0.70
社交媒体：0.40
普通论坛 / 评论：0.25
```

聚合公式：

```python
sentiment_positive = sum(confidence_i * final_weight_i for label_i == "positive") / sum(final_weight_i)
sentiment_negative = sum(confidence_i * final_weight_i for label_i == "negative") / sum(final_weight_i)
sentiment_neutral  = sum(confidence_i * final_weight_i for label_i == "neutral")  / sum(final_weight_i)
```

综合情绪方向：

```python
sentiment_net = sentiment_positive - sentiment_negative
sentiment_strength = abs(sentiment_net)
```

说明：

```text
sentiment_positive / sentiment_negative 是基础变量，不一定直接作为最终标签。
它们会参与 short_term_emotion、panic_selling、weak_sentiment、stop_loss_plan 等标签计算。
```

### 8.3 LLM / 本地模型输出建议

LLM 或本地模型不要直接输出最终 sentiment 分数，而是输出可组合的中间信号：

```json
{
  "event_importance": 0.75,
  "policy_relevance": 0.60,
  "policy_importance": 0.70,
  "llm_herding_signal": 0.50,
  "llm_institutional_signal": 0.65,
  "evidence": [
    "近期公告可能影响市场预期",
    "文本中出现机构调研和基金持仓相关信息"
  ]
}
```

### 8.4 建议配置项

| 参数 | 默认建议 | 说明 |
|---|---|---|
| `attention_center` | `1.5` | 关注度 Robust Z-Score 的明显命中中心 |
| `attention_scale` | `0.8` | 关注度分数上升速度 |
| `sentiment_half_life_hours` | 按报告类型配置 | 文本情绪时间衰减半衰期 |
| `news_half_life_hours` | 按报告类型配置 | 新闻事件时间衰减半衰期 |
| `policy_half_life_hours` | 按报告类型配置 | 政策事件时间衰减半衰期 |
| `source_weights` | 公告 `1.00`，主流财经新闻 `0.80`，研报摘要 `0.70`，社交媒体 `0.40`，论坛评论 `0.25` | 不同信息源可信度 |
| `short_term_emotion_weights` | `{sentiment_positive:0.35, price_rise:0.25, volume_expand:0.25, market_attention_rise:0.15}` | 短线情绪升温的具名权重 |
| `panic_selling_weights` | `{price_drop:0.30, volume_expand:0.25, sentiment_negative:0.25, break_recent_low:0.20}` | 恐慌抛售的具名权重 |
| `sector_rotation_weights` | `{sector_relative_strength:0.40, sector_breadth:0.30, sector_volume_expand:0.30}` | 板块轮动的具名权重 |
| `weak_sentiment_weights` | `{sentiment_negative:0.35, price_drop:0.25, low_attention:0.20, rebound_weak:0.20}` | 情绪偏弱的具名权重 |
| `herding_effect_weights` | `{market_attention_rise:0.30, high_turnover:0.25, volume_expand:0.25, llm_herding_signal:0.20}` | 羊群效应的具名权重 |

## 9. risk_strategy：风险 / 策略标签

### 9.1 推荐方式总表

`risk_strategy` 模块主要由前面 6 个模块的标签组合生成。

LLM 可以参与解释和少数语义风险识别，但不建议完全由 LLM 直接打最终分。

| 标签 | 中文含义 | 推荐方式 | 是否需要 LLM / 模型 | 公式 / 计算方式 |
|---|---|---|---|---|
| `chase_high_risk` | 追高风险 | 规则为主 | 不需要 | `score = weighted_sum({"price_rise": price_rise, "near_recent_high": near_recent_high, "volume_expand": volume_expand, "high_turnover": high_turnover, "short_term_emotion": short_term_emotion}, chase_high_risk_weights)` |
| `false_breakout_risk` | 假突破风险 | 规则为主 | 不需要 | `score = weighted_sum({"breakout": breakout, "close_weak": close_weak, "upper_shadow": upper_shadow, "volume_expand": volume_expand}, false_breakout_risk_weights)` |
| `liquidity_risk` | 流动性风险 | 规则为主 | 不需要 | `score = weighted_sum({"low_turnover": low_turnover, "low_volume": low_volume, "wide_spread": wide_spread}, liquidity_risk_weights)` |
| `drawdown_risk` | 回撤风险 | 规则为主 | 不需要 | `score = weighted_sum({"near_recent_high": near_recent_high, "volatility": volatility, "price_rise": price_rise, "support_distance": support_distance}, drawdown_risk_weights)` |
| `valuation_trap_risk` | 估值陷阱风险 | 混合 | 建议用 LLM | `score = low_valuation * fundamental_negative` |
| `overheated_risk` | 过热风险 | 规则为主 | 不需要 | `score = weighted_sum({"price_rise": price_rise, "volume_expand": volume_expand, "high_turnover": high_turnover, "short_term_emotion": short_term_emotion}, overheated_risk_weights)` |
| `risk_control` | 风险控制 | 规则为主 | 不需要 | `score = 1 - product(1 - r for r in [chase_high_risk, false_breakout_risk, liquidity_risk, drawdown_risk, valuation_trap_risk, overheated_risk])` |
| `position_control` | 仓位控制 | 规则为主 | 不需要 | `score = weighted_sum({"risk_control": risk_control, "volatility": volatility, "uncertainty": uncertainty}, position_control_weights)` |
| `wait_confirm` | 等待确认 | 混合 | 可用 LLM 辅助解释 | `score = 1 - product(1 - s for s in [breakout * (1 - volume_expand), short_term_emotion * (1 - trend_confirmed), policy_driven * (1 - price_confirmed), sentiment_conflict])` |
| `observe_next_day` | 观察次日表现 | 规则为主 | 不需要 | `score = max(news_driven, policy_driven, market_attention_rise, volume_spike, large_intraday_move) * (1 - trend_confirmed)` |
| `avoid_emotional_trade` | 避免情绪交易 | 规则为主 | 不需要 | `score = 1 - product(1 - s for s in [short_term_emotion, panic_selling, herding_effect, overheated_risk])` |
| `take_profit_plan` | 止盈计划 | 规则为主 | 不需要 | `score = weighted_sum({"price_rise": price_rise, "near_recent_high": near_recent_high, "overheated_risk": overheated_risk, "drawdown_risk": drawdown_risk}, take_profit_plan_weights)` |
| `stop_loss_plan` | 止损计划 | 规则为主 | 不需要 | `score = weighted_sum({"break_recent_low": break_recent_low, "downtrend": downtrend, "sentiment_negative": sentiment_negative, "drawdown_risk": drawdown_risk}, stop_loss_plan_weights)` |

### 9.2 risk_strategy 关键变量

| 变量 | 含义 | 公式 / 来源 |
|---|---|---|
| `low_volume` | 低成交 | `clamp((1 - volume_ratio_5d) / 0.5)` |
| `wide_spread` | 买卖价差大 | `clamp(spread_pct / spread_threshold)` |
| `low_valuation` | 低估值强度 | `max(low_pe, low_pb, high_dividend)` |
| `fundamental_negative` | 基本面负面强度 | LLM / 本地模型 / 财务规则输出 |
| `trend_confirmed` | 趋势确认度 | 例如 `MA5 > MA10 > MA20` 且 `current_price > MA5` 时较高 |
| `price_confirmed` | 价格确认度 | 突破后是否站稳关键位 |
| `sentiment_conflict` | 情绪冲突 | 可由规则或 LLM 判断，例如情绪偏正但价格下跌、消息利好但成交不跟随 |
| `uncertainty` | 不确定性 | `weighted_sum({"wait_confirm": wait_confirm, "sentiment_conflict": sentiment_conflict, "policy_uncertainty": policy_uncertainty, "news_uncertainty": news_uncertainty}, uncertainty_weights)` |

### 9.3 risk_strategy 说明

```text
risk_strategy 不是独立原始数据模块，而是综合模块。
它应该读取 asset、price、volume、trend、valuation、sentiment 的输出，再生成风险和策略标签。
```

### 9.4 建议配置项

| 参数 | 默认建议 | 说明 |
|---|---|---|
| `chase_high_risk_weights` | `{price_rise:0.25, near_recent_high:0.25, volume_expand:0.20, high_turnover:0.15, short_term_emotion:0.15}` | 追高风险的具名权重 |
| `false_breakout_risk_weights` | `{breakout:0.35, close_weak:0.25, upper_shadow:0.20, volume_expand:0.20}` | 假突破风险的具名权重 |
| `liquidity_risk_weights` | `{low_turnover:0.40, low_volume:0.30, wide_spread:0.30}` | 流动性风险的具名权重 |
| `drawdown_risk_weights` | `{near_recent_high:0.30, volatility:0.25, price_rise:0.25, support_distance:0.20}` | 回撤风险的具名权重 |
| `overheated_risk_weights` | `{price_rise:0.30, volume_expand:0.25, high_turnover:0.25, short_term_emotion:0.20}` | 过热风险的具名权重 |
| `position_control_weights` | `{risk_control:0.50, volatility:0.25, uncertainty:0.25}` | 仓位控制的具名权重 |
| `take_profit_plan_weights` | `{price_rise:0.30, near_recent_high:0.25, overheated_risk:0.25, drawdown_risk:0.20}` | 止盈计划的具名权重 |
| `stop_loss_plan_weights` | `{break_recent_low:0.35, downtrend:0.25, sentiment_negative:0.20, drawdown_risk:0.20}` | 止损计划的具名权重 |
| `spread_threshold` | 按资产类型配置 | 买卖价差风险阈值 |
| `support_distance_threshold` | 按资产类型配置 | 距离支撑位的风险阈值 |
| `uncertainty_weights` | `{wait_confirm:0.30, sentiment_conflict:0.25, policy_uncertainty:0.25, news_uncertainty:0.20}` | 不确定性的具名权重 |

## 10. 哪些标签需要 LLM / 本地模型

### 10.1 强烈建议用 LLM 或本地模型

| 标签 | 原因 |
|---|---|
| `policy_driven` | 政策对不同资产、行业影响不同，不能只靠关键词 |
| `news_driven` | 需要理解事件重要性，以及行情是否可能由事件触发 |
| `institutional_behavior` | 机构行为常隐藏在研报、公告、基金持仓、调研信息里 |
| `valuation_trap_risk` | 低估值是否是陷阱，需要理解财报、公告、新闻中的基本面风险 |
| `fundamental_risk` | 需要识别盈利下滑、现金流恶化、资产质量问题、监管处罚等语义信息 |
| `herding_effect` | 文本里的从众、抱团、抢筹、净买入、净卖出等语义适合模型识别 |

### 10.2 建议用 FinBERT / 本地情绪模型

| 标签 / 基础变量 | 原因 |
|---|---|
| `sentiment_positive` | 基础正面情绪变量，供 `short_term_emotion` 等公式使用 |
| `sentiment_negative` | 基础负面情绪变量，供 `panic_selling`、`weak_sentiment` 等公式使用 |

## 11. 精度优先的落地建议

如果目标是“数值准度优先”，推荐按下面优先级实现：

```text
第一优先级：历史分位数 percentile_rank
第二优先级：Robust Z-Score + Sigmoid
第三优先级：数据缺失处理，不使用固定阈值硬算
```

原因：

```text
固定阈值容易受行业和资产类型影响。
例如银行股 3% 换手率可能已经很高，但小盘题材股 3% 并不高。
历史分位数可以自动适配不同标的自身特征。
如果历史、行业和同资产类型分布都不足，固定阈值给出的分数通常只是伪精确。
```

推荐实现方式：

```python
# 1. 有该标的历史数据时，优先用该标的历史分位
score = percentile_rank(current_value, asset_history_values)

# 2. 没有足够历史数据时，用行业分布
score = percentile_rank(current_value, industry_values)

# 3. 历史、行业和同资产类型数据都不足时，不硬算
score = None
evidence.append("该标签依赖的分布数据不足，未生成有效分数")
```

数据不足时：

```text
如果某个公式依赖的数据缺失，不要让 LLM 猜数值。
该标签可以不生成，或者生成 score = 0，并在 evidence 中记录数据缺失原因。
```

最终建议：

```text
规则负责数值稳定性；
FinBERT / 本地模型负责文本情绪；
LLM 负责政策、事件、机构行为、基本面风险等语义信号；
所有模型输出都尽量转成中间变量，再由公式合成最终标签分数。
```
