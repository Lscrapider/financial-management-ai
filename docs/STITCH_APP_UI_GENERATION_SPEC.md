# Stitch 移动应用界面生成规格

项目 ID：`2874790038311897756`

用途：这份规格用于在 Stitch MCP 中生成移动应用界面。输入依据是 `docs/APP_PAGE_FUNCTION_DESIGN.md`、当前 Web 代码和产品设计系统。目标不是做营销页，而是生成覆盖全部功能点的产品型应用原型。

语言要求：所有生成屏幕中的可见文案必须为简体中文，不出现英文单词或英文短语。允许股票代码、数字、百分比、401/403 等业务必要符号。

## 1. 设计方向

产品定位：个人投资研究驾驶舱。

用户每天打开应用，需要先判断：

- 哪些观察标的有异动。
- 哪些布控提醒已经越界或接近阈值。
- 哪些报告、知识库材料、文字识别入库任务需要处理。
- 哪里可以继续查看行情、报告、知识证据和智能研究助手建议。

视觉风格：

- 深色研究工作面，背景使用 `#14161a`，主面板使用 `#1c1e23`，二级面板使用 `#24272d`。
- 命令蓝 `#006be6` 只用于主按钮、当前选中、链接和激活状态。
- A 股语义色：上涨/强势用红 `#dc4446`，下跌/弱势用绿 `#57d188`。
- Amber `#efbd48` 用于待处理、接近阈值、处理中。
- 卡片半径 8px，最多 12px。不要 24px 以上圆角。
- 以列表、分段控件、筛选 chips、底部抽屉、状态标签、表单控件为主。
- 不使用营销式 hero、装饰渐变、玻璃拟态、彩色噪声、交易门户式强饱和布局。

移动结构：

- 设备：Mobile，390px 宽。
- 底部主导航：工作台、行情、观察、研究、我的。
- 管理员专属能力放在“我的 > 管理中心”或屏幕内的管理入口，包含知识库和系统管理。
- 每个长任务都要有状态、进度、失败原因、刷新动作。
- 每个列表都要有搜索、筛选或空态。

在 Stitch 中先创建或更新设计系统：

```text
创建一个移动优先的深色产品设计系统，名称为“金融研究驾驶舱”，用于个人投资研究助手。

字体使用苹方或系统无衬线字体。整体是克制的深色研究工作面：背景 #14161a，主面板 #1c1e23，抬升面板 #24272d，弱化控件 #2e3033，边框 #36363a，主文字 #f2f2f2，次级文字 #aeb4bd。

命令蓝 #006be6 只用于主操作、选中导航、链接和激活控件。A 股上涨、正向价格变化和风险/破坏性状态使用 #dc4446；下跌、负向价格变化和非行情语义的成功状态使用 #57d188；待处理、接近阈值和需要关注状态使用 #efbd48。

所有屏幕为 390px 宽的移动应用屏幕。页面边距 16px，使用 8px 网格和 8px 卡片圆角，数据行紧凑但可读，数字按表格方式对齐。使用产品型控件：分段控件、标签页、筛选器、标签、可搜索选择器、开关、滑杆、进度步骤、底部抽屉和详情抽屉。不要装饰渐变、玻璃拟态、超大首屏、交易门户式拥挤布局，也不要营销式智能文案。
```

## 4. Stitch 生成总提示

```text
为“金融管理研究助手”设计一套完整的移动应用界面，这是个人投资研究驾驶舱。使用“金融研究驾驶舱”深色设计系统。

生成 12 个移动端屏幕，全部 390px 宽，覆盖下方所有功能。这是产品型界面，不是营销站点。整体要紧凑、精准、克制，适合用户每天重复做投资研究。

全局规则：
- 所有可见界面文案必须为简体中文，不出现英文单词或英文短语；允许股票代码、数字、百分比、401/403。
- 底部导航：工作台、行情、观察、研究、我的。
- 管理员专属的知识库和系统功能出现在“我的 > 管理中心”以及带有管理员标识的屏幕中。
- 蓝色只用于选中状态和主操作。
- A 股上涨、正向行情和风险/破坏性状态用红色；下跌和负向行情用绿色。
- 每个屏幕都要有真实的金融数据、时间戳、状态标签、筛选器和可操作控件。
- 不暗示智能研究助手能替用户做投资决策；它只提供证据、报告解读和上下文。

屏幕：
1. 登录与权限
展示角色分段控件“管理员/普通用户”、账号和密码输入框、滑块校验状态、记住账号、登录按钮、注册入口。验证码登录、扫码登录、忘记密码展示为不可用的次级入口，并说明“暂未接入”。登录成功目标显示为“回到投资工作台”。

2. 投资工作台
展示顶部问候和刷新按钮、驾驶舱信号“关注”、三个指标标签：观察池 18、越界 2、报告生成中 1。展示今日行动列表、提醒优先级、观察池异动列表、资产分布、提醒覆盖、最新报告卡片。包含智能研究助手悬浮入口。

3. 行情中心
顶部标签为股票、指数、可转债。展示搜索、市场筛选、可排序列表、选中标的详情面板、价格、涨跌幅、成交量、成交额、行情字段、走势图周期分段控件“分时/日线/周线/月线”、走势预览和“更多行情”操作。包含指数卡片跳转和可转债评级/转股溢价字段。

4. 观察风控
展示分组标签“核心持仓/低估观察/转债”。列表包含标的最新价、涨跌幅、买入价、持仓、持仓市值、浮动盈亏和备注。包含“添加标的”底部抽屉摘要、编辑标的控件、阈值提醒区域、启用开关、阈值输入、接近/触发状态，以及带管理员标识的“手动检查”按钮。

5. 报告研究
展示报告标的列表，包含标的类型筛选、状态标签、最新版本、报告数量和摘要。包含“新建报告”底部抽屉：配置分组/模板选择、标的类型/标的搜索选择、报告类型、召回条数、日线/周线/月线数量上限、核心参数滑杆，操作为“创建报告任务”和“保存为配置”。轮询状态使用“待处理/检索中/生成中/成功/失败”。

6. 报告详情工作台
展示报告详情，包括标的、模型、版本、生成时间、报告正文分节和引用证据标签。下方用组件堆栈替代桌面拖拽网格：报告主体、走势图、盘口数据、详情数据。包含配置组件、选择历史报告和导出文档操作。

7. 投资心理画像
展示问卷进度 7/10、分组问题、可选答案、重置和保存按钮。结果面板包含建议强度、波动情绪、决策风格、操作节奏、回答偏好、持仓心态，并显示“会影响智能研究助手建议口径”。

8. 知识库材料
明确这是“材料检索”，不是“报告生成”。模式分段控件为“按标的/自然语言”。按标的模式展示配置分组/模板、标的类型、标的搜索、报告场景、日线/周线/月线窗口、召回条数、参数覆盖。自然语言模式有文本域。主按钮为“检索材料”。结果状态包含任务状态，按场景分组，并提供“筛选场景/筛选标签/知识库名称”筛选。结果卡片展示来源文件名、片段序号、中文场景标签、中文标签、综合分、语义分、原文。不要单独出现“检索理解”这一行。

9. 知识库管理
展示知识库概览指标：文档数、知识条目、总文本量、最近更新。展示按场景和标签统计的分布条形图。展示片段浏览器，包含来源类型、场景、标签、文档名筛选，列表行、选中片段详情、原文、元数据、编辑原文和编辑标签操作。标记为仅管理员可见。

10. 知识入库
展示文字识别导入任务列表、上传操作、任务统计、六阶段流程：文字识别、清洗、质量校验、场景打标、向量入库、完成，并展示状态和进度。包含人工复核视图、段落列表、页面影像证据、保存草稿和提交复核。包含手动知识导入：标题、多段原文编辑器、保存草稿、提交入库。标记为仅管理员可见。

11. 系统管理
展示管理员管理中心，标签为系统监控、调用用量、标的配置、数据同步。系统监控展示访问量、用户数、智能调用数、同步状态。调用用量展示费用、请求数、用量分类、来源/阶段/模型/用户筛选和日志详情。标的配置展示新增股票/可转债以及危险的物理删除确认。数据同步展示股票/指数/可转债同步卡片、全量同步、单标的补分时/日线，以及长任务轮询状态。

12. 我的与智能研究助手
展示个人资料摘要、基础信息编辑、安全设置、修改密码、邮件通知开关、可见的管理员入口。包含智能研究助手全屏对话：上下文标签为当前页、自选池、近 20 条会话、心理画像、知识库材料，展示流式消息、输入框，以及基于证据的安全回答风格。
```

## 6. 生成后验收点

- Stitch 项目中能看到新的移动应用屏幕，且不是旧的理财/资产管理泛化页面。
- 屏幕使用深色研究驾驶舱风格，而不是浅色 `Pro-Fin Digital` 泛金融模板。
- 知识库材料屏幕必须有“按标的/自然语言”两种检索方式，并展示场景、标签、知识库名称筛选。
- 报告研究屏幕和知识库材料屏幕必须区分：前者创建报告任务，后者检索材料且不生成报告。
- 管理员专属功能必须有权限标识或放在管理入口。
- 所有可见界面文案必须为简体中文，不出现英文单词或英文短语。
- 所有功能文档页面都能在覆盖矩阵中找到对应 UI。
