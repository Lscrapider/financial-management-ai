

# 标的标签计算规则

本文档用于定义 report 生成阶段中，当前标的 `currentScenes` 的 7 大类标签计算方式。

核心目标：

```text
chunk 入库时已经有 7 大类标签；
当前标的分析时，也要生成同一套 7 大类标签，并且每个标签带 0~1 的分数；
后续根据当前标的标签分数，动态召回更匹配的知识 chunk。
```

---

## 1. 通用约定

### 1.1 分数范围

所有标签分数统一归一化到 `0 ~ 1`：

```python
score = 0      # 不命中 / 无明显信号
score = 0.5    # 中等强度
score = 1.0    # 强信号
```

统一使用：

```python
def clamp(x):
    return min(max(x, 0), 1)
```

### 1.1.1 适用标的约定

标签白名单中的 `适用标的` 是标签语义的一部分。当前标的计算时，只有目标资产类型匹配时才产出资产专属标签；chunk 入库时也必须按适用标的打标签，避免可转债和基金的专属概念污染股票检索。

| 适用标的 | 说明 |
|---|---|
| 通用 | 股票、指数、可转债、基金都可复用，例如趋势、回撤、仓位控制。 |
| 股票 | 只用于股票及股票细分场景，例如 PE、PB、股息率、银行股。 |
| 指数 | 只用于指数及指数行情场景。 |
| 可转债 | 只用于可转债，例如转股溢价、强赎、债底、剩余规模。 |
| 基金 | 只用于基金、ETF、LOF、债基、货基、QDII 等基金场景。 |

跨资产同名概念必须使用资产前缀区分：

```text
convertible_high_premium：可转债转股溢价高
fund_premium：基金场内溢价
```

这样 chunk 重新打标签后，RAG 检索不会把可转债溢价经验召回到基金溢价报告中。

### 1.2 精度优先的归一化函数

为了提高数值准度，不建议使用固定阈值线性映射作为标签兜底。`clamp` 只用于最终边界保护，核心评分优先使用历史分位数、Robust Z-Score、Sigmoid 和时间衰减。

#### 1.2.1 Sigmoid 平滑映射

适合处理涨跌幅、成交量倍率、换手率、波动率等连续指标。

```python
import math

def sigmoid_score(x, center, scale):
    return 1 / (1 + math.exp(-(x - center) / scale))
```

含义：

```text
center：开始明显命中的中心点。
scale：分数上升速度，越小越敏感。
```

例如：

```python
price_rise = sigmoid_score(change_pct, center=2.0, scale=1.2)
```

这表示涨幅接近 2% 时开始明显命中，涨幅越大分数越接近 1。

#### 1.2.2 历史分位数评分

适合 PE、PB、成交额、换手率、波动率这类不同资产差异很大的指标。

```python
def percentile_rank(value, history_values):
    valid_values = [v for v in history_values if v is not None]
    if not valid_values:
        return None
    return sum(1 for v in valid_values if v <= value) / len(valid_values)
```

例如：

```python
high_turnover = percentile_rank(turnover_rate, turnover_rate_history_250d)
low_pb = 1 - percentile_rank(pb, pb_history_5y)
```

含义：

```text
当前换手率处于历史 90% 分位，则 high_turnover = 0.90。
当前 PB 处于历史 20% 分位，则 low_pb = 0.80。
```

历史分位数类指标不再使用固定阈值兜底。数据来源按标的历史分布、行业分布、同资产类型分布逐级降级；仍然不足时，该标签不生成或生成 `score = 0`，并在 `evidence` 中记录数据缺失原因。

#### 1.2.3 Robust Z-Score 异动评分

适合判断“是否异常”，例如成交量突然放大、新闻数量突然增加、日内波动突然变大。

```python
def robust_z_score(value, median, mad):
    return (value - median) / (1.4826 * mad + 1e-9)

def z_to_score(z, center=1.5, scale=1.0):
    return sigmoid_score(z, center=center, scale=scale)
```

例如：

```python
volume_z = robust_z_score(current_volume, median_volume_60d, mad_volume_60d)
volume_spike = z_to_score(volume_z, center=1.8, scale=0.7)
```

含义：

```text
Robust Z-Score 比普通均值比值更抗极端值影响。
适合真实行情中成交量偶尔异常巨大的情况。
```

#### 1.2.4 时间衰减权重

适合新闻、公告、社交讨论、研报等文本信号。

```python
time_weight = math.exp(-hours_since_publish / half_life_hours)
```

建议：

```text
短线报告：half_life_hours = 24 ~ 72。
中期报告：half_life_hours = 168，也就是 7 天。
```

#### 1.2.5 多证据合成公式

多个信号同时支持一个标签时，不建议永远简单平均。可以使用加权平均，也可以使用 noisy-or。

加权平均：

```python
score = sum(w_i * s_i for w_i, s_i in signals) / sum(w_i for w_i, s_i in signals)
```

noisy-or：

```python
score = 1 - product(1 - w_i * s_i for w_i, s_i in signals)
```

如果代码实现：

```python
def noisy_or(scores):
    result = 1.0
    for score in scores:
        result *= (1 - clamp(score))
    return 1 - result
```

加权求和：

```python
def weighted_sum(signals, weights):
    valid_items = [
        (signal_name, score)
        for signal_name, score in signals.items()
        if score is not None and signal_name in weights
    ]
    weight_sum = sum(weights[signal_name] for signal_name, score in valid_items)
    if weight_sum <= 0:
        return 0
    return clamp(
        sum(weights[signal_name] * score for signal_name, score in valid_items) / weight_sum
    )
```

区别：

```text
加权平均：适合多个证据都需要综合考虑的标签。
noisy-or：适合任意一个强证据出现就应该明显提高分数的标签，例如 news_driven、policy_driven、institutional_behavior、risk_control。
```

### 1.3 模块输出结构

当前 Python 回调给 Java 的 `currentScenesPayload.currentScenes` 中，每个模块统一输出：

```json
{
  "score": 0.74,
  "level": "high",
  "direction": "positive",
  "tags": {
    "price_rise": 0.80,
    "near_recent_high": 0.75
  },
  "evidence": [
    "当日涨幅超过上涨强度阈值，price_rise 标签触发",
    "当前价格处于近 20 日区间高位，near_recent_high 标签触发"
  ],
  "queryText": "价格分析，价格上涨、接近近期高位。当日涨幅超过上涨强度阈值，当前价格处于近 20 日区间高位。"
}
```

注意：

- `module` 是 Python 内部 `SceneModuleResult` 字段，不进入当前回调给 Java 的 `currentScenes` 模块 payload。
- 下文各模块章节中的单模块 JSON 示例如果出现 `module` 字段，仅用于说明模块名；最终 `currentScenesPayload.currentScenes` 不包含该字段。
- `queryText` 会进入 `currentScenesPayload`，用于后续 retrieval embedding；Java 构建最终 LLM 输入时会裁剪掉该字段。
- `risk_strategy` 模块在 `currentScenes` 输出中的 key 为 `riskStrategy`，Java 接收时兼容 `risk_strategy` 别名。
- `trend` 模块额外输出 `periodTrends.daily`、`periodTrends.weekly`、`periodTrends.monthly`，用于报告里的日 K、周 K、月 K 趋势分析。

`evidence` 固定为字符串数组，不额外嵌套 `metrics`、`reason` 等对象字段。

`evidence` 文案要求：

- 客观描述本次标签触发依据，不写投资建议和主观判断。
- 粒度细到可支持 RAG 召回和 LLM 后续组合，例如说明“哪个标签触发”和“触发依据属于价格、量能、估值、情绪代理还是风险子信号”。
- 组合标签只描述本次实际达到阈值的子信号，避免把未触发的信号写进证据。
- 对行情代理情绪标签，需要说明“基于行情代理信号计算”，不要暗示使用了新闻、公告或外部文本情绪。

### 1.4 模块总分计算

模块总分不建议简单取平均，因为很多标签不会同时出现。

第一版推荐：

```python
module_score = 0.7 * max(tag_scores) + 0.3 * avg(non_zero_tag_scores)
```

不再使用纯 `max(tag_scores)`，避免单个高分标签把整个模块分数拉得过高。

对于 `risk_strategy` 这类多证据累积型模块，以及 `news_driven`、`policy_driven` 这类多证据累积型标签，可以优先使用 noisy-or。

### 1.5 可配置参数约定

标签计算应拆成两层：

```text
计算公式：保持稳定，由系统内置；
公式参数：允许按默认配置、资产类型配置、用户配置逐层覆盖。
```

也就是说，页面可以让用户调整阈值、权重和敏感度，但不建议让普通用户直接修改公式本身。这样既能保持标签含义稳定，也能支持不同资产类型和不同用户风格。

不是所有公式常量都需要配置化。只有影响用户风格、资产适配、信息源信任度和模型敏感度的参数建议进入配置；标签定义型常量应作为系统内置规则固定，避免同一个标签在不同用户之间语义漂移。

#### 1.5.1 参数覆盖优先级

推荐覆盖顺序：

```text
用户自定义参数 > 资产类型参数 > 系统默认参数
```

示例：

```text
普通股票高换手优先使用该标的历史换手率分布；
银行股可以覆盖为银行行业换手率分布；
用户可以通过参数面板调整阈值、敏感度和权重。
```

如果某个参数没有用户配置，使用资产类型配置；资产类型也没有配置时，使用系统默认值。

#### 1.5.3 建议抽取的参数类型

| 参数类型 | 示例 | 用途 |
|---|---|---|
| 阈值参数 | `low_price_threshold`、`gap_threshold`、`pullback_threshold`、`support_distance_threshold` | 控制什么程度算命中 |
| Sigmoid 参数 | `price_rise_center`、`price_rise_scale`、`volume_expand_center`、`volume_expand_scale`、`volume_spike_center`、`volume_spike_scale` | 控制连续指标的敏感度 |
| 权重参数 | `market_proxy_emotion_weights`、`market_proxy_panic_weights`、`chase_high_risk_weights`、`overheated_risk_weights` | 控制多个信号合成一个标签时的侧重点 |

#### 1.5.4 页面配置建议

页面不建议一次暴露全部参数。第一版可以分为两层：

```text
普通配置：选择参数模板；
高级配置：调整具体阈值、权重、敏感度和来源权重。
```

参数模板可以先提供：

```text
稳健型：降低短线情绪和追涨信号权重，提高风险控制权重；
平衡型：使用系统默认参数；
短线敏感型：提高涨跌幅、放量、关注度和消息驱动的敏感度；
低估值偏好：提高低 PE、低 PB、高股息相关标签权重；
风险优先：提高回撤、流动性、假突破、估值陷阱相关标签权重；
银行股专用：优先使用银行行业换手率、PB、股息率分布，提高资产质量相关权重；
可转债专用：单独配置换手率、溢价率、剩余规模、强赎相关参数。
```

#### 1.5.5 参数快照

每次生成 `currentScenes` 时，保存本次实际使用的参数快照。

```json
{
  "profileCode": "system_recommended",
  "userOverrides": {
    "asset_type": "bank_stock",
    "price_config": {
      "price_rise_center": 2.0,
      "price_rise_scale": 1.2
    }
  }
}
```

参数快照用于后续解释、复盘和排查：同一个标的在不同用户参数下，标签分数可能不同。

#### 1.5.6 权重参数使用方式

权重参数必须在公式中显式引用，不应只出现在配置表中。

推荐使用具名权重，避免只写 `0.25,0.25,0.20` 导致顺序不清楚。

```json
{
  "chase_high_risk_weights": {
    "price_rise": 0.25,
    "near_recent_high": 0.25,
    "volume_expand": 0.20,
    "high_turnover": 0.15,
    "short_term_emotion": 0.15
  }
}
```

公式统一写成：

```python
score = weighted_sum(
    signals={
        "price_rise": price_rise,
        "near_recent_high": near_recent_high,
        "volume_expand": volume_expand,
        "high_turnover": high_turnover,
        "short_term_emotion": short_term_emotion
    },
    weights=chase_high_risk_weights
)
```

如果用户只调整其中一个权重，当前 Python 会按有效权重总和做加权平均，不要求保存配置时把权重和归一到 `1.0`。

#### 1.5.7 分布数据来源约定

当前 report 不暴露分布来源配置。成交量、换手率、交易关注度等分位数类指标由 Python 基于当前标的行情历史直接计算；如后续引入行业或资产类型横向分布，再新增对应配置项。

如果历史分布数据不足：

```text
不使用固定阈值硬算；
该标签不生成，或生成 score = 0；
evidence 中记录“分布数据不足”。
```

## 2. 基础变量计算

后续 7 个模块都会复用这些基础变量。

| 变量 | 含义 | 公式 / 计算方式 |
|---|---|---|
| `change_pct` | 当日涨跌幅百分比 | 行情接口直接获取，例如 `3.5` 表示上涨 3.5% |
| `price_rise` | 上涨强度 | `sigmoid_score(change_pct, center=price_rise_center, scale=price_rise_scale)`，仅当 `change_pct > 0` 生效 |
| `price_drop` | 下跌强度 | `sigmoid_score(abs(change_pct), center=price_drop_center, scale=price_drop_scale)`，仅当 `change_pct < 0` 生效 |
| `price_move` | 价格异动强度 | `sigmoid_score(abs(change_pct), center=price_move_center, scale=price_move_scale)` |
| `volume_ratio_5d` | 当前成交量 / 近 5 日均量 | `current_volume / avg_volume_5d` |
| `volume_ratio_20d` | 当前成交量 / 近 20 日均量 | `current_volume / avg_volume_20d` |
| `volume_expand` | 放量强度 | `z_to_score(robust_z_score(current_volume, median_volume_60d, mad_volume_60d), center=volume_expand_center, scale=volume_expand_scale)` |
| `volume_shrink` | 缩量强度 | `1 - percentile_rank(current_volume, volume_history_60d)` |
| `volume_spike` | 爆量强度 | `z_to_score(robust_z_score(current_volume, median_volume_60d, mad_volume_60d), center=volume_spike_center, scale=volume_spike_scale)` |
| `high_turnover` | 高换手强度 | `percentile_rank(turnover_rate, turnover_rate_distribution)` |
| `low_turnover` | 低换手强度 | `1 - high_turnover` |
| `position_20d` | 当前价格在近 20 日区间的位置 | `(current_price - low_20d) / (high_20d - low_20d)` |
| `near_recent_high` | 接近近期高位 | `clamp((position_20d - 0.8) / 0.2)` |
| `near_recent_low` | 接近近期低位 | `clamp((0.2 - position_20d) / 0.2)` |
| `breakout` | 突破前高 | `1 if current_price > prev_high_20d else 0` |
| `break_recent_low` | 跌破前低 | `1 if current_price < prev_low_20d else 0` |
| `close_weak` | 收盘弱 | `clamp((intraday_high - close_price) / (intraday_high - intraday_low))` |
| `upper_shadow` | 上影线强度 | `clamp((high_price - max(open_price, close_price)) / (high_price - low_price))` |
| `large_intraday_move` | 日内大波动 | `clamp((high_price - low_price) / prev_close / 0.08)` |
| `volatility` | 波动率强度 | `percentile_rank(atr_14 / current_price, volatility_history_250d)` |
| `support_distance` | 距离支撑位强度 | `clamp(((current_price - support_price) / current_price) / support_distance_threshold)` |
| `trading_attention` | 交易关注度代理值 | `log1p(amount) * (1 + turnover_rate / 100) * (1 + amplitude / 100)` |
| `trading_attention_rise` | 交易关注度上升倍数 | `trading_attention_today / avg(trading_attention_20d)` |
| `market_attention_rise` | 市场关注度上升强度 | `sigmoid_score(trading_attention_rise, center=attention_rise_center, scale=attention_rise_scale)` |
| `low_attention` | 市场关注度偏低强度 | `clamp((1 - trading_attention_rise) / low_attention_scale)` |

### 2.1 可转债专属基础变量

可转债专属变量来自转债行情、转债条款、正股行情和正股 K 线。变量放入 Python message 的资产专属数据块 `assetSpecificData.convertibleBond`，不要和股票估值字段混在同一个 `valuationData` 中。

当前 Java 数据来源：

```text

convertible_bond_daily_valuation：

convertible_bond_share：
  份额变化和期末剩余规模。

bond_quote_snapshot：
```

| 变量 | 含义 | 公式 / 来源 |

| `maturity_days` | 距离到期天数 | `maturity_date - today` |
| `redeem_status` | 强赎状态 | 未触发 / 接近触发 / 已触发 / 已公告等；数据不足时为 `DATA_INSUFFICIENT`，计算端不得当成已触发 |

| `underlying_change_pct` | 正股涨跌幅 | 正股最新行情 |
| `underlying_trend_score` | 正股趋势强度 | 正股 K 线运行通用 trend 计算后得到 |
| `stock_bond_linkage` | 正股联动强度 | 转债涨跌与正股涨跌方向、幅度和近期相关性综合计算 |

## 4. price：价格位置标签

### 4.1 推荐方式

`price` 模块全部建议用规则计算，不建议用 LLM。

| 标签 | 中文含义 | 适用标的 | 推荐方式 | 公式 / 计算方式 |
|---|---|---|---|---|
| `price_rise` | 价格上涨 | 通用 | 规则 | `score = sigmoid_score(change_pct, center=price_rise_center, scale=price_rise_scale)`，仅当 `change_pct > 0` 生效 |
| `price_drop` | 价格下跌 | 通用 | 规则 | `score = sigmoid_score(abs(change_pct), center=price_drop_center, scale=price_drop_scale)`，仅当 `change_pct < 0` 生效 |
| `sideways` | 横盘 | 通用 | 规则 | `range_pct_20d = (high_20d - low_20d) / low_20d`；`score = clamp((0.08 - range_pct_20d) / 0.08)` |
| `near_recent_high` | 接近近期高位 | 通用 | 规则 | `position_20d = (current_price - low_20d) / (high_20d - low_20d)`；`score = clamp((position_20d - 0.8) / 0.2)` |
| `near_recent_low` | 接近近期低位 | 通用 | 规则 | `score = clamp((0.2 - position_20d) / 0.2)` |
| `breakout` | 突破 | 通用 | 规则 | `score = 1.0 if current_price > prev_high_20d else 0.0` |
| `pullback` | 回调 | 通用 | 规则 | `score = uptrend * clamp((recent_high - current_price) / recent_high / pullback_threshold)` |
| `gap_up` | 跳空高开 | 股票 / 指数 / 可转债 | 规则 | `score = clamp((open_price - prev_high) / prev_close / gap_threshold)`，仅当 `open_price > prev_high` 生效 |
| `gap_down` | 跳空低开 | 股票 / 指数 / 可转债 | 规则 | `score = clamp((prev_low - open_price) / prev_close / gap_threshold)`，仅当 `open_price < prev_low` 生效 |
| `convertible_high_price_risk` | 转债高价风险 | 可转债 | 规则 | `score = sigmoid_score(bond_price, center=high_price_threshold, scale=high_price_scale)` |
| `convertible_low_price_defensive` | 转债低价防御 | 可转债 | 规则 | `score = clamp((low_price_threshold - bond_price) / low_price_threshold)`，债底或到期收益率有支撑时增强 |
| `fund_nav_rise` | 基金净值上涨 | 基金 | 规则 | 基于基金净值涨幅或净值收益率序列计算，公式同 `price_rise` |
| `fund_nav_drop` | 基金净值下跌 | 基金 | 规则 | 基于基金净值跌幅或净值收益率序列计算，公式同 `price_drop` |

### 4.2 说明

```text
price 模块描述“当前价格状态”。
它不负责判断后续策略，后续策略由 risk_strategy 模块组合生成。
```

| 参数 | 默认建议 | 说明 |
|---|---|---|

| `price_rise_scale` | `1.2` | `price_rise` 分数上升速度 |
| `price_drop_center` | `2.0` | 跌幅接近多少时开始明显命中 `price_drop` |
| `price_drop_scale` | `1.2` | `price_drop` 分数上升速度 |
| `price_move_center` | `2.0` | 涨跌异动接近多少时开始明显命中 `price_move` |
| `price_move_scale` | `1.2` | `price_move` 分数上升速度 |
| `pullback_threshold` | 按资产类型配置 | 回调幅度阈值 |
| `gap_threshold` | 按资产类型配置 | 跳空幅度阈值 |

## 5. volume：成交量 / 换手标签

### 5.1 推荐方式

`volume` 模块主要基于成交量、成交额、换手率、量价关系，建议用规则计算。

| 标签 | 中文含义 | 适用标的 | 推荐方式 | 公式 / 计算方式 |
|---|---|---|---|---|
| `volume_expand` | 放量 | 股票 / 指数 / 可转债 / ETF / LOF | 规则 | `score = z_to_score(robust_z_score(current_volume, median_volume_60d, mad_volume_60d), center=volume_expand_center, scale=volume_expand_scale)` |

| `fund_share_growth` | 基金份额增长 | 基金 | 规则 | `score = percentile_rank(fund_share_change, same_fund_type_share_change_distribution)` |
| `fund_share_shrink` | 基金份额缩水 | 基金 | 规则 | `score = percentile_rank(-fund_share_change, same_fund_type_share_change_distribution)` |

| `volume_expand_scale` | `0.8` | 放量分数上升速度 |
| `volume_spike_center` | `1.8` | 爆量 Robust Z-Score 的明显命中中心 |
| `volume_spike_scale` | `0.7` | 爆量分数上升速度 |

## 6. trend：趋势结构标签

### 6.1 推荐方式

`trend` 模块主要用 K 线结构和均线关系判断，第一版建议规则计算。

| 标签 | 中文含义 | 适用标的 | 推荐方式 | 公式 / 计算方式 |
|---|---|---|---|---|
| `uptrend` | 上升趋势 | 通用 | 规则 | `MA5 > MA10 > MA20` 时基础分 `0.8`；若 `current_price > MA5` 加 `0.1`；若 `current_price > MA20` 加 `0.1` |
| `downtrend` | 下降趋势 | 通用 | 规则 | `MA5 < MA10 < MA20` 时基础分 `0.8`；若 `current_price < MA5` 加 `0.1`；若 `current_price < MA20` 加 `0.1` |
| `range_bound` | 区间震荡 | 通用 | 规则 | `range_pct_20d = (high_20d - low_20d) / low_20d`；`score = clamp((0.10 - range_pct_20d) / 0.10)` |
| `rebound` | 反弹 | 通用 | 规则 | 当前 report 基于前一段下跌和最新一段上涨的路径特征计算，不读取配置阈值 |
| `pullback` | 回调 | 通用 | 规则 | `score = uptrend * clamp((recent_high - current_price) / recent_high / pullback_threshold)`，强调上涨趋势内的回踩而不是趋势破坏 |
| `repair` | 修复 | 通用 | 规则 | `score = max(downtrend, weak_trend) * stabilization_score * slow_recovery_score`，用于下跌或弱势后的企稳、跌幅收敛和缓慢回升 |
| `trend_reversal` | 趋势反转 | 通用 | 规则为主 | `score = max(downtrend_to_up_signal, uptrend_to_down_signal)`；例如均线开始金叉 / 死叉、价格重新站上 / 跌破 MA20 |
| `breakout_from_range` | 横盘突破 | 通用 | 规则 | 当前 report 基于最新收盘价突破前一窗口高点的强度计算，不读取配置权重 |
| `breakdown_from_range` | 区间破位 | 通用 | 规则 | `score = range_bound_recent * support_break_score * downside_confirm_score`，用于跌破平台、跌破箱体或跌破关键支撑 |
| `continuation` | 趋势延续 | 通用 | 规则 | `score = previous_trend_score * current_same_direction_score`，用于前序趋势和当前趋势方向一致 |
| `turn_weak` | 转弱 | 通用 | 规则 | `score = uptrend_recent * max(momentum_fade, support_break, ma_bearish_shift)`，用于上涨后动能衰减、上攻乏力或趋势开始走弱 |
| `turn_strong` | 转强 | 通用 | 规则 | `score = max(downtrend_recent, range_bound_recent) * max(momentum_recover, center_lift, ma_bullish_shift)`，用于下跌或横盘后重心抬升、弱势改善 |
| `failed_breakout` | 突破失败 | 通用 | 规则 | `score = breakout_recent * close_weak * upper_shadow` |
| `fund_nav_uptrend` | 基金净值上升趋势 | 基金 | 规则 | 基于基金净值或场内基金价格序列运行趋势计算，语义等同基金净值趋势向上 |
| `fund_nav_downtrend` | 基金净值下降趋势 | 基金 | 规则 | 基于基金净值或场内基金价格序列运行趋势计算，语义等同基金净值趋势向下 |

### 6.2 说明

```text
trend 看的是一段时间结构，不是单日涨跌。
例如 price_rise 高，只能说明今天涨了，不一定说明 uptrend 已经形成。
```

| 参数 | 默认建议 | 说明 |
|---|---|---|
| `pullback_threshold` | 按资产类型配置 | 上升趋势内回调幅度阈值 |

## 7. valuation：估值 / 基本面标签

### 7.1 推荐方式

`valuation` 模块当前已实现 PE / PB / 股息率和估值修复的规则计算。`valuation_trap` 和 `fundamental_risk` 作为预留标签保留，当前 report 代码不产出这两个标签。

| 标签 | 中文含义 | 适用标的 | 推荐方式 | 公式 / 计算方式 |
|---|---|---|---|---|
| `low_pe` | 低 PE | 股票 | 规则 | `score = 1 - percentile_rank(pe, pe_distribution)` |
| `high_pe` | 高 PE | 股票 | 规则 | `score = percentile_rank(pe, pe_distribution)` |
| `low_pb` | 低 PB | 股票 | 规则 | `score = 1 - percentile_rank(pb, pb_distribution)` |
| `high_pb` | 高 PB | 股票 | 规则 | `score = percentile_rank(pb, pb_distribution)` |
| `high_dividend` | 高股息 | 股票 | 规则 | `score = percentile_rank(dividend_yield, dividend_yield_distribution)` |
| `valuation_repair` | 估值修复 | 股票 | 规则为主 | `score = low_valuation * price_rise * uptrend` |
| `valuation_trap` | 低估值陷阱 | 股票 | 预留 | 当前 report 不产出；未来可基于 `low_valuation * fundamental_negative` 扩展 |
| `fundamental_risk` | 基本面风险 | 股票 / 可转债 / 基金 | 预留 | 股票看财报基本面；可转债看正股基本面和发行主体信用；基金看持仓、基金公司、债券信用暴露等 |
| `convertible_low_premium` | 转股低溢价 | 可转债 | 规则 | `score = clamp((premium_low_threshold - premium_rate) / premium_low_threshold)` |
| `convertible_high_premium` | 转股高溢价 | 可转债 | 规则 | `score = sigmoid_score(premium_rate, center=premium_high_threshold, scale=premium_high_scale)` |
| `convertible_premium_compression` | 溢价压缩 | 可转债 | 规则 | `score = percentile_rank(-premium_rate_change, premium_compression_history)`，无历史时使用近 N 日溢价率下降幅度 |
| `convertible_premium_expansion` | 溢价扩张 | 可转债 | 规则 | `score = percentile_rank(premium_rate_change, premium_expansion_history)`，无历史时使用近 N 日溢价率上升幅度 |
| `convertible_debt_floor_support` | 债底支撑 | 可转债 | 规则 | `score = clamp(1 - (bond_price - pure_bond_value) / debt_floor_distance_threshold)` |
| `convertible_high_ytm` | 到期收益率较高 | 可转债 | 规则 | `score = sigmoid_score(ytm, center=high_ytm_threshold, scale=ytm_scale)` |
| `convertible_low_ytm` | 到期收益率较低或为负 | 可转债 | 规则 | `score = clamp((low_ytm_threshold - ytm) / abs(low_ytm_threshold))` |
| `convertible_high_conversion_value` | 转股价值较高 | 可转债 | 规则 | `score = percentile_rank(conversion_value, conversion_value_history)`，无历史时结合 `conversion_value >= 100` 和正股趋势 |
| `fund_premium` | 基金场内溢价 | 基金 | 规则 | `score = sigmoid_score(fund_premium_rate, center=premium_threshold, scale=premium_scale)`，主要用于 ETF / LOF |
| `fund_discount` | 基金场内折价 | 基金 | 规则 | `score = sigmoid_score(fund_discount_rate, center=discount_threshold, scale=discount_scale)`，主要用于 ETF / LOF |
| `fund_high_fee` | 费率偏高 | 基金 | 规则 | `score = percentile_rank(expense_ratio, same_fund_type_fee_distribution)` |
| `fund_large_scale` | 基金规模较大 | 基金 | 规则 | `score = percentile_rank(fund_scale, same_fund_type_scale_distribution)` |
| `fund_small_scale` | 基金规模较小 | 基金 | 规则 | `score = 1 - percentile_rank(fund_scale, same_fund_type_scale_distribution)` |
| `fund_tracking_error` | 跟踪误差 | 基金 | 规则 | `score = sigmoid_score(tracking_error, center=tracking_error_threshold, scale=tracking_error_scale)` |
| `fund_high_drawdown` | 基金高回撤 | 基金 | 规则 | `score = percentile_rank(abs(fund_drawdown), same_fund_type_drawdown_distribution)` |
| `fund_stable_nav` | 净值稳定 | 基金 | 规则 | `score = 1 - percentile_rank(fund_volatility, same_fund_type_volatility_distribution)` |

### 7.2 关键基础变量

```python
low_valuation = max(low_pe, low_pb, high_dividend)
```

可转债估值不使用 PE / PB / 股息率。可转债估值模块应单独读取 `premium_rate`、`conversion_value`、`pure_bond_value`、`ytm`、`bond_price` 等变量。

基金估值不使用股票 PE / PB。基金估值模块应按基金类型分流：ETF / LOF 关注溢折价、跟踪误差和流动性；主动基金关注费率、回撤、规模和持仓集中；债基关注久期、信用暴露和净值稳定性。

以下为预留扩展公式，当前 report 代码不执行：

```python
fundamental_negative = LLM_or_local_model(
    financial_report_summary,
    announcements,
    news
)
```

### 7.3 说明

```text
低 PE / 低 PB 不一定等于机会。
当前代码不计算基本面恶化语义信号，因此不会自动提高 valuation_trap 和 fundamental_risk。
```

## 8. sentiment：情绪 / 异动标签

### 8.1 推荐方式总表

`sentiment` 模块不是只输出一个情绪值，而是输出多个标签及其分数。

| 标签 | 中文含义 | 适用标的 | 推荐方式 | 是否需要 LLM / 模型 | 公式 / 计算方式 |
|---|---|---|---|---|---|
| `market_attention_rise` | 关注度上升 | 通用 | 规则为主 | 不需要 | `score = sigmoid_score(trading_attention_today / avg(trading_attention_20d), center=attention_rise_center, scale=attention_rise_scale)` |
| `short_term_emotion` | 短线情绪升温 | 股票 / 指数 / 可转债 / ETF / LOF | 规则为主 | 不需要 | `score = market_proxy_emotion` |
| `panic_selling` | 恐慌抛售 | 通用 | 规则为主 | 不需要 | `score = market_proxy_panic` |
| `news_driven` | 消息驱动 | 通用 | 预留 | 建议用 LLM | 当前 report 不产出，不参与任何标签计算 |
| `policy_driven` | 政策驱动 | 通用 | 预留 | 建议用 LLM | 当前 report 不产出，不参与任何标签计算 |
| `sector_rotation` | 板块轮动 | 股票 / 指数 / 基金 | 预留 | 不一定需要 | 当前 report 不产出；未来可基于板块相对强度、板块广度和板块放量计算 |
| `weak_sentiment` | 情绪偏弱 | 通用 | 规则为主 | 不需要 | `score = market_proxy_weak` |
| `herding_effect` | 羊群效应 / 从众行为 | 股票 / 指数 / 可转债 / 基金 | 规则为主 | 不需要 | `score = market_proxy_herding`；基金可叠加份额快速流入 / 流出 |
| `institutional_behavior` | 机构 / 基金行为 | 股票 / 指数 / 基金 | 预留 | 建议用 LLM 或结构化资金数据 | 当前 report 不产出，不参与任何标签计算 |
| `convertible_stock_linkage` | 正股联动 | 可转债 | 规则为主 | 不需要 | `score = weighted_sum({underlying_change_pct, underlying_trend_score, stock_bond_linkage}, weights)` |
| `convertible_independent_strength` | 转债独立走强 | 可转债 | 规则为主 | 不需要 | `score = max(price_rise * (1 - stock_bond_linkage), convertible_premium_expansion)` |
| `fund_flow_in` | 基金资金流入 | 基金 | 规则为主 | 不需要 | `score = max(fund_share_growth, etf_net_inflow_rank)` |
| `fund_flow_out` | 基金资金流出 | 基金 | 规则为主 | 不需要 | `score = max(fund_share_shrink, etf_net_outflow_rank)` |

### 8.2 行情代理情绪变量

当前代码仅使用稳定行情数据计算 sentiment，不使用 AkShare、人气榜、新闻、公告、研报等外部数据。

交易关注度代理值：

```text
trading_attention = log1p(amount) * (1 + turnover_rate / 100) * (1 + amplitude / 100)
trading_attention_rise = trading_attention_today / avg(trading_attention_20d)
market_attention_rise = sigmoid_score(trading_attention_rise, center=attention_rise_center, scale=attention_rise_scale)
low_attention = clamp((1 - trading_attention_rise) / low_attention_scale)
```

说明：

```text
amount 使用成交额，单位必须前后一致；
turnover_rate 和 amplitude 使用百分比值，例如 3.5 表示 3.5%；
log1p 用于避免成交额过大导致分数被成交额单一变量支配。
```

行情代理分：

```text
market_proxy_emotion = weighted_sum({
  price_rise,
  volume_expand,
  high_turnover,
  market_attention_rise
}, market_proxy_emotion_weights)

market_proxy_panic = weighted_sum({
  price_drop,
  volume_expand,
  break_recent_low,
  close_weak
}, market_proxy_panic_weights)

market_proxy_weak = weighted_sum({
  price_drop,
  volume_shrink,
  low_turnover,
  close_weak,
  low_attention
}, market_proxy_weak_weights)

market_proxy_herding = weighted_sum({
  high_turnover,
  volume_expand,
  market_attention_rise
}, market_proxy_herding_weights)
```

当前代码不计算外部文本 / 热度增强项：

```text
sentiment_positive 不计算
sentiment_negative 不计算
text_attention_boost 不计算
text_negative_boost 不计算
external_sentiment_confidence 固定视为 0
```

evidence 中应说明“基于行情代理信号计算，未使用外部文本情绪”。

### 8.3 预留：FinBERT 情绪聚合

FinBERT 每条文本通常输出：

```json
{"label": "positive", "score": 0.87}
```

为了让情绪分数更稳定，推荐使用“置信度 × 时间衰减 × 来源权重”的加权方式。

```python
time_weight = exp(-hours_since_publish / half_life_hours)
source_weight = get_source_weight(source_type)
final_weight = time_weight * source_weight
```

建议来源权重：

```text
公告 / 财报：1.00
主流财经新闻：0.80
研报摘要：0.70
社交媒体：0.40
普通论坛 / 评论：0.25
```

聚合公式：

```python
sentiment_positive = sum(confidence_i * final_weight_i for label_i == "positive") / sum(final_weight_i)
sentiment_negative = sum(confidence_i * final_weight_i for label_i == "negative") / sum(final_weight_i)
sentiment_neutral  = sum(confidence_i * final_weight_i for label_i == "neutral")  / sum(final_weight_i)
```

综合情绪方向：

```python
sentiment_net = sentiment_positive - sentiment_negative
sentiment_strength = abs(sentiment_net)
```

说明：

```text
当前代码不接入 FinBERT / 本地情绪模型，因此不计算 sentiment_positive / sentiment_negative / sentiment_neutral。
这些变量只作为后续增强项保留，不参与当前标签计算。
```

### 8.4 预留：LLM / 本地模型输出建议

LLM 或本地模型不要直接输出最终 sentiment 分数，而是输出可组合的中间信号：

```json
{
  "event_importance": 0.75,
  "policy_relevance": 0.60,
  "policy_importance": 0.70,
  "llm_herding_signal": 0.50,
  "llm_institutional_signal": 0.65,
  "evidence": [
    "近期公告可能影响市场预期",
    "文本中出现机构调研和基金持仓相关信息"
  ]
}
```

### 8.5 建议配置项

| 参数 | 默认建议 | 说明 |
|---|---|---|
| `attention_rise_center` | `1.5` | `trading_attention_rise` 达到 1.5 倍附近时认为关注度明显升温 |
| `attention_rise_scale` | `0.4` | 交易关注度分数上升速度 |
| `low_attention_scale` | `0.5` | 交易关注度低于 20 日均值时的低关注度归一化尺度 |

## 9. risk_strategy：风险 / 策略标签

### 9.1 推荐方式总表

`risk_strategy` 模块主要由前面 6 个模块的标签组合生成。

当前 report 代码基于规则和行情代理信号生成风险策略标签；LLM 只在最终报告生成阶段使用，不参与这些标签打分。

| 标签 | 中文含义 | 适用标的 | 推荐方式 | 是否需要 LLM / 模型 | 公式 / 计算方式 |

| `convertible_putback_risk` | 回售相关风险 | 可转债 | 规则为主 | 不需要 | 基于 `putback_status`、回售触发距离、到期时间和价格位置计算 |
| `convertible_low_rating_risk` | 低评级风险 | 可转债 | 规则为主 | 不需要 | 评级落入 `low_rating_levels` 时按等级映射，例如 A 以下更高 |
| `convertible_small_balance_risk` | 剩余规模过小风险 | 可转债 | 规则为主 | 不需要 | `score = clamp((small_balance_threshold - remaining_size) / small_balance_threshold)` |
| `convertible_liquidity_risk` | 转债流动性风险 | 可转债 | 规则为主 | 不需要 | `score = noisy_or([low_turnover, low_volume, convertible_small_balance_risk])` |
| `fund_tracking_deviation_risk` | 基金跟踪偏离风险 | 基金 | 规则为主 | 不需要 | `score = fund_tracking_error`，仅指数基金 / ETF / LOF 使用 |
| `fund_concentration_risk` | 基金持仓集中风险 | 基金 | 规则为主 | 不需要 | 基于行业集中度、前十大持仓集中度或单一风格暴露计算 |
| `fund_credit_risk` | 债基信用风险 | 基金 | 规则为主 | 不一定需要 | 基于低评级债暴露、信用债占比、持仓主体风险计算 |
| `fund_duration_risk` | 债基久期风险 | 基金 | 规则为主 | 不需要 | 基于 `bond_duration` 与同类分布分位数计算 |
| `fund_qdii_fx_risk` | QDII 汇率风险 | 基金 | 规则为主 | 不需要 | 基于 QDII 类型、汇率波动、海外资产敞口计算 |
| `fund_liquidity_risk` | 基金流动性风险 | 基金 | 规则为主 | 不需要 | ETF / LOF 看成交额和溢折价，场外基金看限购限赎、巨额赎回和清盘风险 |

### 9.2 risk_strategy 关键变量

| 变量 | 含义 | 公式 / 来源 |
|---|---|---|
| `low_volume` | 低成交 | `clamp((1 - volume_ratio_5d) / 0.5)` |
| `low_valuation` | 低估值强度 | `max(low_pe, low_pb, high_dividend)` |
| `fundamental_negative` | 基本面负面强度 | 预留变量，当前 report 不计算 |
| `trend_confirmed` | 趋势确认度 | 例如 `MA5 > MA10 > MA20` 且 `current_price > MA5` 时较高 |
| `price_confirmed` | 价格确认度 | 突破后是否站稳关键位 |
| `sentiment_conflict` | 情绪冲突 | `max(short_term_emotion * price_drop, panic_selling * price_rise)` |
| `uncertainty` | 不确定性 | 当前 report 使用 `sentiment_conflict`；`policy_uncertainty`、`news_uncertainty` 为预留变量 |

### 9.3 risk_strategy 说明

```text
risk_strategy 不是独立原始数据模块，而是综合模块。
它应该读取 asset、price、volume、trend、valuation、sentiment 的输出，再生成风险和策略标签。
```

## 10. 哪些标签需要 LLM / 本地模型

### 10.1 强烈建议用 LLM 或本地模型

| 标签 | 原因 |
|---|---|
| `policy_driven` | 政策对不同资产、行业影响不同，不能只靠关键词 |
| `news_driven` | 需要理解事件重要性，以及行情是否可能由事件触发 |
| `institutional_behavior` | 机构行为常隐藏在研报、公告、基金持仓、调研信息里 |
| `valuation_trap_risk` | 低估值是否是陷阱，需要理解财报、公告、新闻中的基本面风险 |
| `fundamental_risk` | 需要识别盈利下滑、现金流恶化、资产质量问题、监管处罚等语义信息 |
| `herding_effect` | 基础交易拥挤度可以用规则代理，但文本里的从众、抱团、抢筹、净买入、净卖出等语义适合模型增强 |

### 10.2 建议用 FinBERT / 本地情绪模型

| 标签 / 基础变量 | 原因 |
|---|---|
| `sentiment_positive` | 基础正面情绪变量，供 `short_term_emotion` 等公式使用 |
| `sentiment_negative` | 基础负面情绪变量，供 `panic_selling`、`weak_sentiment` 等公式使用 |
| `sentiment_neutral` | 基础中性情绪变量，用于判断消息是否缺少方向性 |
| `text_attention_boost` | 文本正面情绪、外部热度、消息驱动的增强分支 |
| `text_negative_boost` | 文本负面情绪、负面新闻、负面政策的增强分支 |

### 10.3 规则更稳定，不建议用 LLM

| 标签 |
|---|
| `asset` 下大部分资产类型标签 |
| `price` 下所有价格位置标签 |
| `volume` 下所有成交量 / 换手标签 |
| `trend` 下大部分趋势结构标签 |
| `low_pe`、`high_pe`、`low_pb`、`high_pb`、`high_dividend` |
| `market_attention_rise` |
| `short_term_emotion` 的行情代理分支 |
| `panic_selling` 的行情代理分支 |
| `weak_sentiment` 的行情代理分支 |
| `herding_effect` 的行情代理分支 |
| `sector_rotation` |
| `chase_high_risk` |
| `false_breakout_risk` |
| `liquidity_risk` |
| `drawdown_risk` |
| `overheated_risk` |
| `risk_control` |
| `position_control` |
| `observe_next_day` |
| `avoid_emotional_trade` |
| `take_profit_plan` |
| `stop_loss_plan` |

## 11. 精度优先的落地建议

如果目标是“数值准度优先”，推荐按下面优先级实现：

```text
第一优先级：历史分位数 percentile_rank
第二优先级：Robust Z-Score + Sigmoid
第三优先级：数据缺失处理，不使用固定阈值硬算
```

原因：

```text
固定阈值容易受行业和资产类型影响。
例如银行股 3% 换手率可能已经很高，但小盘题材股 3% 并不高。
历史分位数可以自动适配不同标的自身特征。
如果历史、行业和同资产类型分布都不足，固定阈值给出的分数通常只是伪精确。
```

推荐实现方式：

```python
# 1. 有该标的历史数据时，优先用该标的历史分位
score = percentile_rank(current_value, asset_history_values)

# 2. 没有足够历史数据时，用行业分布
score = percentile_rank(current_value, industry_values)

# 3. 历史、行业和同资产类型数据都不足时，不硬算
score = None
evidence.append("该标签依赖的分布数据不足，未生成有效分数")
```

数据不足时：

```text
如果某个公式依赖的数据缺失，不要让 LLM 猜数值。
该标签可以不生成，或者生成 score = 0，并在 evidence 中记录数据缺失原因。
```

后续增强时的建议：

```text
规则负责数值稳定性；
FinBERT / 本地模型负责文本情绪；
LLM 负责政策、事件、机构行为、基本面风险等语义信号；
所有模型输出都尽量转成中间变量，再由公式合成最终标签分数。
```
