# 标的标签计算规则

本文档用于定义 report 生成阶段中，当前标的 `currentScenes` 的 7 大类标签计算方式。

核心目标：

```text
chunk 入库时已经有 7 大类标签；
当前标的分析时，也要生成同一套 7 大类标签，并且每个标签带 0~1 的分数；
后续根据当前标的标签分数，动态召回更匹配的知识 chunk。
```

---

## 1. 通用约定

### 1.1 分数范围

所有标签分数统一归一化到 `0 ~ 1`：

```python
score = 0      # 不命中 / 无明显信号
score = 0.5    # 中等强度
score = 1.0    # 强信号
```

统一使用：

```python
def clamp(x):
    return min(max(x, 0), 1)
```

### 1.2 精度优先的归一化函数

为了提高数值准度，不建议所有指标都直接使用固定阈值线性映射。第一版可以保留 `clamp` 作为兜底，但在数据充足时，优先使用历史分位数、Robust Z-Score、Sigmoid 和时间衰减。

#### 1.2.1 Sigmoid 平滑映射

适合处理涨跌幅、成交量倍率、换手率、波动率等连续指标。

```python
import math

def sigmoid_score(x, center, scale):
    return 1 / (1 + math.exp(-(x - center) / scale))
```

例如：

```python
price_rise = sigmoid_score(change_pct, center=PRICE_SIGMOID_CENTER, scale=PRICE_SIGMOID_SCALE)
```

#### 1.2.2 历史分位数评分

适合 PE、PB、成交额、换手率、波动率这类不同资产差异很大的指标。

```python
def percentile_rank(value, history_values):
    valid_values = [v for v in history_values if v is not None]
    if not valid_values:
        return None
    return sum(1 for v in valid_values if v <= value) / len(valid_values)
```

例如：

```python
high_turnover = percentile_rank(turnover_rate, turnover_rate_history_250d)
low_pb = 1 - percentile_rank(pb, pb_history_5y)
```

#### 1.2.3 Robust Z-Score 异动评分

适合判断“是否异常”，例如成交量突然放大、新闻数量突然增加、日内波动突然变大。

```python
def robust_z_score(value, median, mad):
    return (value - median) / (1.4826 * mad + EPSILON)

def z_to_score(z, center=1.5, scale=1.0):
    return sigmoid_score(z, center=center, scale=scale)
```

#### 1.2.4 时间衰减权重

适合新闻、公告、社交讨论、研报等文本信号。

```python
time_weight = math.exp(-hours_since_publish / half_life_hours)
```

建议：

```text
短线报告：half_life_hours = 24 ~ 72。
中期报告：half_life_hours = 168，也就是 7 天。
```

#### 1.2.5 多证据合成公式

加权平均：

```python
score = sum(w_i * s_i for w_i, s_i in signals) / sum(w_i for w_i, s_i in signals)
```

noisy-or：

```python
score = 1 - product(1 - w_i * s_i for w_i, s_i in signals)
```

代码实现：

```python
def noisy_or(scores):
    result = 1.0
    for score in scores:
        result *= (1 - clamp(score))
    return 1 - result
```

说明：

```text
加权平均：适合多个证据都需要综合考虑的标签。
noisy-or：适合任意一个强证据出现就应该明显提高分数的标签，例如 news_driven、policy_driven、institutional_behavior、risk_control。
```

### 1.3 模块输出结构

```json
{
  "module": "price",
  "score": 0.74,
  "level": "high",
  "direction": "positive",
  "tags": {
    "price_rise": 0.80,
    "near_recent_high": 0.75
  },
  "evidence": [
    "当日涨幅较明显",
    "当前价格接近近期高位"
  ]
}
```

### 1.4 模块总分计算

第一版推荐：

```python
module_score = max(tag_scores)
```

如果希望多个标签同时命中时提高模块重要性，可以使用：

```python
module_score = MODULE_SCORE_MAX_WEIGHT * max(tag_scores) + MODULE_SCORE_AVG_WEIGHT * avg(non_zero_tag_scores)
```

对于 `risk_strategy`、`news_driven`、`policy_driven` 这类多证据累积型模块，可以优先使用 noisy-or。

## 2. 默认常量配置

本节集中定义文档中所有公式会用到的默认常量。后续代码实现时，建议将这些常量抽取为统一配置类，例如 `SceneScoreConfig`，避免公式中散落硬编码数值。

### 2.1 通用分数配置

| 常量名 | 默认值 | 说明 |
|---|---:|---|
| `SCORE_MIN` | `0.0` | 标签最低分，表示不命中或无明显信号 |
| `SCORE_MID` | `0.5` | 中等强度信号 |
| `SCORE_MAX` | `1.0` | 强信号 |
| `EPSILON` | `1e-9` | 防止除零，例如 MAD、区间宽度、权重和为 0 的情况 |
| `MODULE_SCORE_MAX_WEIGHT` | `0.70` | 模块总分中，最高标签分数的权重 |
| `MODULE_SCORE_AVG_WEIGHT` | `0.30` | 模块总分中，非零标签平均分的权重 |

### 2.2 Sigmoid 映射参数

| 常量名 | 默认值 | 说明 |
|---|---:|---|
| `PRICE_SIGMOID_CENTER` | `2.0` | 涨跌幅接近 2% 时开始明显命中 |
| `PRICE_SIGMOID_SCALE` | `1.2` | 涨跌幅分数上升速度，越小越敏感 |
| `VOLUME_EXPAND_Z_CENTER` | `1.0` | 成交量高于历史中位数约 1 个 robust z-score 时，开始明显放量 |
| `VOLUME_EXPAND_Z_SCALE` | `0.8` | 放量分数敏感度 |
| `VOLUME_SPIKE_Z_CENTER` | `1.8` | 成交量高于历史中位数约 1.8 个 robust z-score 时，开始认为爆量 |
| `VOLUME_SPIKE_Z_SCALE` | `0.7` | 爆量分数敏感度 |
| `ATTENTION_Z_CENTER` | `1.5` | 新闻 / 讨论数量明显高于历史水平时，开始认为关注度上升 |
| `ATTENTION_Z_SCALE` | `0.8` | 关注度上升分数敏感度 |

### 2.3 价格位置与波动阈值

| 常量名 | 默认值 | 说明 |
|---|---:|---|
| `POSITION_HIGH_THRESHOLD` | `0.8` | 当前价格位于近 20 日区间 80% 以上时，开始认为接近近期高位 |
| `POSITION_LOW_THRESHOLD` | `0.2` | 当前价格位于近 20 日区间 20% 以下时，开始认为接近近期低位 |
| `SIDEWAYS_RANGE_THRESHOLD` | `0.08` | 近 20 日振幅小于 8% 时，偏横盘 |
| `RANGE_BOUND_THRESHOLD` | `0.10` | 近 20 日振幅小于 10% 时，偏区间震荡 |
| `LARGE_INTRADAY_MOVE_THRESHOLD` | `0.08` | 日内振幅达到 8% 时，认为日内波动较大 |
| `DEFAULT_GAP_THRESHOLD` | `0.03` | 跳空高开 / 低开的简化阈值，默认 3% |
| `DEFAULT_PULLBACK_THRESHOLD` | `0.08` | 从近期高点回撤约 8% 时，认为回调较明显 |
| `DEFAULT_REBOUND_THRESHOLD` | `0.08` | 从近期低点反弹约 8% 时，认为反弹较明显 |

### 2.4 成交量与换手率阈值

| 常量名 | 默认值 | 说明 |
|---|---:|---|
| `VOLUME_EXPAND_RATIO_BASE` | `1.0` | 成交量超过均量时，开始认为放量 |
| `VOLUME_EXPAND_RATIO_RANGE` | `2.0` | 简化公式中，成交量达到约 3 倍均量时，放量分接近 1 |

| `VOLUME_DRY_UP_RATIO_THRESHOLD` | `0.5` | 成交量低于 20 日均量 50% 时，认为成交枯竭 |
| `ORDINARY_STOCK_TURNOVER_THRESHOLD` | `0.10` | 普通股票高换手兜底阈值，10% |

### 2.5 估值阈值

| 常量名 | 默认值 | 说明 |
|---|---:|---|

| `PB_LOW_RANGE` | `0.5` | 用于低 PB 简化公式：`score = clamp((PB_LOW_REFERENCE - pb) / PB_LOW_RANGE)`。默认 `PB_LOW_REFERENCE = 1.0`，`PB_LOW_RANGE = 0.5`，因此 PB = 0.5 时低 PB 分接近 1 |
| `HIGH_DIVIDEND_THRESHOLD` | `0.06` | 股息率达到 6% 时，高股息分接近 1 |

## 3. 基础变量计算

后续 7 个模块都会复用这些基础变量。

| 变量 | 含义 | 公式 / 计算方式 |
|---|---|---|
| `change_pct` | 当日涨跌幅百分比 | 行情接口直接获取，例如 `3.5` 表示上涨 3.5% |
| `price_rise` | 上涨强度 | 优先：`sigmoid_score(change_pct, center=PRICE_SIGMOID_CENTER, scale=PRICE_SIGMOID_SCALE)`；简化：`clamp(change_pct / 5)`，仅当 `change_pct > 0` 生效 |
| `price_drop` | 下跌强度 | 优先：`sigmoid_score(abs(change_pct), center=PRICE_SIGMOID_CENTER, scale=PRICE_SIGMOID_SCALE)`；简化：`clamp(abs(change_pct) / 5)`，仅当 `change_pct < 0` 生效 |
| `price_move` | 价格异动强度 | 优先：`sigmoid_score(abs(change_pct), center=PRICE_SIGMOID_CENTER, scale=PRICE_SIGMOID_SCALE)`；简化：`clamp(abs(change_pct) / 5)` |
| `volume_ratio_5d` | 当前成交量 / 近 5 日均量 | `current_volume / avg_volume_5d` |
| `volume_ratio_20d` | 当前成交量 / 近 20 日均量 | `current_volume / avg_volume_20d` |
| `volume_expand` | 放量强度 | 优先：`z_to_score(robust_z_score(current_volume, median_volume_60d, mad_volume_60d), center=VOLUME_EXPAND_Z_CENTER, scale=VOLUME_EXPAND_Z_SCALE)`；简化：`clamp((volume_ratio_5d - VOLUME_EXPAND_RATIO_BASE) / VOLUME_EXPAND_RATIO_RANGE)` |
| `volume_shrink` | 缩量强度 | `clamp((1 - volume_ratio_5d) / VOLUME_SHRINK_RATIO_RANGE)` |
| `volume_spike` | 爆量强度 | 优先：`z_to_score(robust_z_score(current_volume, median_volume_60d, mad_volume_60d), center=VOLUME_SPIKE_Z_CENTER, scale=VOLUME_SPIKE_Z_SCALE)`；简化：`clamp((volume_ratio_20d - VOLUME_EXPAND_RATIO_BASE) / VOLUME_SPIKE_RATIO_RANGE)` |
| `high_turnover` | 高换手强度 | 优先：`percentile_rank(turnover_rate, turnover_rate_history_250d)`；简化：`clamp(turnover_rate / turnover_threshold)` |
| `low_turnover` | 低换手强度 | `1 - high_turnover` |
| `position_20d` | 当前价格在近 20 日区间的位置 | `(current_price - low_20d) / (high_20d - low_20d + EPSILON)` |
| `near_recent_high` | 接近近期高位 | `clamp((position_20d - POSITION_HIGH_THRESHOLD) / (1 - POSITION_HIGH_THRESHOLD))` |
| `near_recent_low` | 接近近期低位 | `clamp((POSITION_LOW_THRESHOLD - position_20d) / POSITION_LOW_THRESHOLD)` |
| `breakout` | 突破前高 | `1 if current_price > prev_high_20d else 0` |
| `break_recent_low` | 跌破前低 | `1 if current_price < prev_low_20d else 0` |
| `close_weak` | 收盘弱 | `clamp((intraday_high - close_price) / (intraday_high - intraday_low + EPSILON))` |
| `upper_shadow` | 上影线强度 | `clamp((high_price - max(open_price, close_price)) / (high_price - low_price + EPSILON))` |
| `large_intraday_move` | 日内大波动 | `clamp((high_price - low_price) / prev_close / LARGE_INTRADAY_MOVE_THRESHOLD)` |
| `volatility` | 波动率强度 | 优先：`percentile_rank(atr_14 / current_price, volatility_history_250d)`；简化：`clamp((atr_14 / current_price) / volatility_threshold)` |
| `support_distance` | 距离支撑位强度 | `clamp(((current_price - support_price) / current_price) / support_distance_threshold)` |

## 5. price：价格位置标签

### 5.1 推荐方式

`price` 模块全部建议用规则计算，不建议用 LLM。

| 标签 | 中文含义 | 推荐方式 | 公式 / 计算方式 |
|---|---|---|---|
| `price_rise` | 价格上涨 | 规则 | 优先：`score = sigmoid_score(change_pct, center=PRICE_SIGMOID_CENTER, scale=PRICE_SIGMOID_SCALE)`；简化：`score = clamp(change_pct / 5)`，仅当 `change_pct > 0` 生效 |
| `price_drop` | 价格下跌 | 规则 | 优先：`score = sigmoid_score(abs(change_pct), center=PRICE_SIGMOID_CENTER, scale=PRICE_SIGMOID_SCALE)`；简化：`score = clamp(abs(change_pct) / 5)`，仅当 `change_pct < 0` 生效 |
| `sideways` | 横盘 | 规则 | `range_pct_20d = (high_20d - low_20d) / (low_20d + EPSILON)`；`score = clamp((SIDEWAYS_RANGE_THRESHOLD - range_pct_20d) / SIDEWAYS_RANGE_THRESHOLD)` |
| `near_recent_high` | 接近近期高位 | 规则 | `position_20d = (current_price - low_20d) / (high_20d - low_20d + EPSILON)`；`score = clamp((position_20d - POSITION_HIGH_THRESHOLD) / (1 - POSITION_HIGH_THRESHOLD))` |
| `near_recent_low` | 接近近期低位 | 规则 | `score = clamp((POSITION_LOW_THRESHOLD - position_20d) / POSITION_LOW_THRESHOLD)` |
| `breakout` | 突破 | 规则 | `score = 1.0 if current_price > prev_high_20d else 0.0` |
| `pullback` | 回调 | 规则 | `score = uptrend * clamp((recent_high - current_price) / recent_high / DEFAULT_PULLBACK_THRESHOLD)` |
| `gap_up` | 跳空高开 | 规则 | `score = clamp((open_price - prev_high) / prev_close / DEFAULT_GAP_THRESHOLD)`，仅当 `open_price > prev_high` 生效 |
| `gap_down` | 跳空低开 | 规则 | `score = clamp((prev_low - open_price) / prev_close / DEFAULT_GAP_THRESHOLD)`，仅当 `open_price < prev_low` 生效 |

## 6. volume：成交量 / 换手标签

### 6.1 推荐方式

`volume` 模块主要基于成交量、成交额、换手率、量价关系，建议用规则计算。

| 标签 | 中文含义 | 推荐方式 | 公式 / 计算方式 |
|---|---|---|---|
| `volume_expand` | 放量 | 规则 | 优先：`score = z_to_score(robust_z_score(current_volume, median_volume_60d, mad_volume_60d), center=VOLUME_EXPAND_Z_CENTER, scale=VOLUME_EXPAND_Z_SCALE)`；简化：`score = clamp((volume_ratio_5d - VOLUME_EXPAND_RATIO_BASE) / VOLUME_EXPAND_RATIO_RANGE)` |
| `volume_shrink` | 缩量 | 规则 | `score = clamp((1 - volume_ratio_5d) / VOLUME_SHRINK_RATIO_RANGE)` |
| `high_turnover` | 高换手 | 规则 | 优先：`score = percentile_rank(turnover_rate, turnover_rate_history_250d)`；简化：`score = clamp(turnover_rate / turnover_threshold)` |
| `low_turnover` | 低换手 | 规则 | `score = 1 - high_turnover` |
| `volume_price_confirm` | 量价配合 | 规则 | `score = max(price_rise * volume_expand, price_drop * volume_shrink)` |
| `volume_price_divergence` | 量价背离 | 规则 | `score = max(price_rise * volume_shrink, volume_expand * close_weak)` |
| `volume_spike` | 成交量突然放大 | 规则 | 优先：`score = z_to_score(robust_z_score(current_volume, median_volume_60d, mad_volume_60d), center=VOLUME_SPIKE_Z_CENTER, scale=VOLUME_SPIKE_Z_SCALE)`；简化：`score = clamp((volume_ratio_20d - VOLUME_EXPAND_RATIO_BASE) / VOLUME_SPIKE_RATIO_RANGE)` |
| `volume_dry_up` | 成交枯竭 | 规则 | `score = clamp((VOLUME_DRY_UP_RATIO_THRESHOLD - volume_ratio_20d) / VOLUME_DRY_UP_RATIO_THRESHOLD)` |

## 7. trend：趋势结构标签

### 7.1 推荐方式

`trend` 模块主要用 K 线结构和均线关系判断，第一版建议规则计算。

| 标签 | 中文含义 | 推荐方式 | 公式 / 计算方式 |
|---|---|---|---|
| `uptrend` | 上升趋势 | 规则 | `MA5 > MA10 > MA20` 时基础分 `0.8`；若 `current_price > MA5` 加 `0.1`；若 `current_price > MA20` 加 `0.1` |
| `downtrend` | 下降趋势 | 规则 | `MA5 < MA10 < MA20` 时基础分 `0.8`；若 `current_price < MA5` 加 `0.1`；若 `current_price < MA20` 加 `0.1` |
| `range_bound` | 区间震荡 | 规则 | `range_pct_20d = (high_20d - low_20d) / (low_20d + EPSILON)`；`score = clamp((RANGE_BOUND_THRESHOLD - range_pct_20d) / RANGE_BOUND_THRESHOLD)` |
| `rebound` | 反弹 | 规则 | `score = downtrend * price_rise * clamp((current_price - recent_low) / recent_low / DEFAULT_REBOUND_THRESHOLD)` |
| `trend_reversal` | 趋势反转 | 规则为主 | `score = max(downtrend_to_up_signal, uptrend_to_down_signal)`；例如均线开始金叉 / 死叉、价格重新站上 / 跌破 MA20 |
| `breakout_from_range` | 横盘突破 | 规则 | `score = range_bound_recent * breakout * (MODULE_SCORE_MAX_WEIGHT + MODULE_SCORE_AVG_WEIGHT * volume_expand)` |
| `failed_breakout` | 突破失败 | 规则 | `score = breakout_recent * close_weak * upper_shadow` |

## 8. valuation：估值 / 基本面标签

### 8.1 推荐方式

`valuation` 模块中的 PE / PB / 股息率建议规则计算；`valuation_trap` 和 `fundamental_risk` 可以用 LLM 或本地模型辅助分析公告、财报、新闻。

| 标签 | 中文含义 | 推荐方式 | 公式 / 计算方式 |
|---|---|---|---|
| `low_pe` | 低 PE | 规则 | 优先：`score = 1 - percentile_rank(pe, pe_history_or_industry_pe_distribution)`；简化：有行业均值时 `score = clamp((industry_avg_pe - pe) / industry_avg_pe)` |
| `high_pe` | 高 PE | 规则 | 优先：`score = percentile_rank(pe, pe_history_or_industry_pe_distribution)`；简化：有行业均值时 `score = clamp((pe - industry_avg_pe) / industry_avg_pe)` |
| `low_pb` | 低 PB | 规则 | 优先：`score = 1 - percentile_rank(pb, pb_history_or_industry_pb_distribution)`；简化：`score = clamp((PB_LOW_REFERENCE - pb) / PB_LOW_RANGE)`，适合银行等 PB 重要行业 |
| `high_pb` | 高 PB | 规则 | 优先：`score = percentile_rank(pb, pb_history_or_industry_pb_distribution)`；简化：`score = clamp((pb - industry_avg_pb) / industry_avg_pb)` |
| `high_dividend` | 高股息 | 规则 | 优先：`score = percentile_rank(dividend_yield, dividend_yield_history_or_industry_distribution)`；简化：`score = clamp(dividend_yield / HIGH_DIVIDEND_THRESHOLD)` |
| `valuation_repair` | 估值修复 | 规则为主 | `score = low_valuation * price_rise * uptrend` |
| `valuation_trap` | 低估值陷阱 | 混合 | `score = low_valuation * fundamental_negative` |
| `fundamental_risk` | 基本面风险 | LLM / 本地模型 / 财务规则 | 从财报、公告、新闻中识别盈利下滑、现金流恶化、资产质量下降、监管处罚、坏账上升等，输出 `0~1` |

## 9. sentiment：情绪 / 异动标签

### 9.1 推荐方式总表

`sentiment` 模块不是只输出一个情绪值，而是输出多个标签及其分数。

| 标签 | 中文含义 | 推荐方式 | 是否需要 LLM / 模型 | 公式 / 计算方式 |
|---|---|---|---|---|
| `market_attention_rise` | 关注度上升 | 规则为主 | 不一定需要 | 优先：`attention_z = robust_z_score(current_24h_mentions, median_daily_mentions_60d, mad_daily_mentions_60d)`；`score = z_to_score(attention_z, center=ATTENTION_Z_CENTER, scale=ATTENTION_Z_SCALE)`；简化：`attention_ratio = current_24h_mentions / avg_daily_mentions_7d`；`score = clamp((attention_ratio - 1) / 2)` |
| `short_term_emotion` | 短线情绪升温 | 混合 | 建议用 FinBERT / 本地模型 | `score = SHORT_TERM_EMOTION_SENTIMENT_POSITIVE_WEIGHT * sentiment_positive + SHORT_TERM_EMOTION_PRICE_RISE_WEIGHT * price_rise + SHORT_TERM_EMOTION_VOLUME_EXPAND_WEIGHT * volume_expand + SHORT_TERM_EMOTION_ATTENTION_WEIGHT * market_attention_rise` |
| `panic_selling` | 恐慌抛售 | 混合 | 建议用 FinBERT / 本地模型 | `score = PANIC_SELLING_PRICE_DROP_WEIGHT * price_drop + PANIC_SELLING_VOLUME_EXPAND_WEIGHT * volume_expand + PANIC_SELLING_SENTIMENT_NEGATIVE_WEIGHT * sentiment_negative + PANIC_SELLING_BREAK_LOW_WEIGHT * break_recent_low` |
| `news_driven` | 消息驱动 | 混合 | 建议用 LLM | 推荐：`score = 1 - product(1 - event_importance_i * time_weight_i * max(price_move, volume_expand) for each event_i)`；简化：`score = event_importance * max(price_move, volume_expand)` |
| `policy_driven` | 政策驱动 | 模型为主 | 建议用 LLM | 推荐：`score = 1 - product(1 - policy_relevance_i * policy_importance_i * time_weight_i for each policy_i)`；简化：`score = policy_relevance * policy_importance` |
| `sector_rotation` | 板块轮动 | 规则为主 | 不一定需要 | `score = SECTOR_ROTATION_RELATIVE_STRENGTH_WEIGHT * sector_relative_strength + SECTOR_ROTATION_BREADTH_WEIGHT * sector_breadth + SECTOR_ROTATION_VOLUME_WEIGHT * sector_volume_expand` |
| `weak_sentiment` | 情绪偏弱 | 混合 | 建议用 FinBERT / 本地模型 | `score = WEAK_SENTIMENT_NEGATIVE_WEIGHT * sentiment_negative + WEAK_SENTIMENT_PRICE_DROP_WEIGHT * price_drop + WEAK_SENTIMENT_LOW_ATTENTION_WEIGHT * low_attention + WEAK_SENTIMENT_REBOUND_WEAK_WEIGHT * rebound_weak` |
| `herding_effect` | 羊群效应 / 从众行为 | 混合 | 建议用 LLM 或关键词模型 | `score = HERDING_ATTENTION_WEIGHT * market_attention_rise + HERDING_HIGH_TURNOVER_WEIGHT * high_turnover + HERDING_VOLUME_EXPAND_WEIGHT * volume_expand + HERDING_LLM_SIGNAL_WEIGHT * llm_herding_signal` |
| `institutional_behavior` | 机构 / 基金行为 | 混合 | 建议用 LLM 或结构化资金数据 | 推荐：`score = 1 - product(1 - s for s in [fund_flow_signal, institutional_news_signal, analyst_report_signal, llm_institutional_signal])`；简化：`score = max(fund_flow_signal, institutional_news_signal, analyst_report_signal, llm_institutional_signal)` |

### 9.2 FinBERT 情绪聚合

FinBERT 每条文本通常输出：

```json
{"label": "positive", "score": 0.87}
```

为了让情绪分数更稳定，推荐使用“置信度 × 时间衰减 × 来源权重”的加权方式。

```python
time_weight = exp(-hours_since_publish / half_life_hours)
source_weight = get_source_weight(source_type)
final_weight = time_weight * source_weight
```

建议来源权重：

```text
公告 / 财报：1.00
主流财经新闻：0.80
研报摘要：0.70
社交媒体：0.40
普通论坛 / 评论：0.25
```

聚合公式：

```python
sentiment_positive = sum(confidence_i * final_weight_i for label_i == "positive") / sum(final_weight_i)
sentiment_negative = sum(confidence_i * final_weight_i for label_i == "negative") / sum(final_weight_i)
sentiment_neutral  = sum(confidence_i * final_weight_i for label_i == "neutral")  / sum(final_weight_i)
```

综合情绪方向：

```python
sentiment_net = sentiment_positive - sentiment_negative
sentiment_strength = abs(sentiment_net)
```

### 9.3 LLM / 本地模型输出建议

LLM 或本地模型不要直接输出最终 sentiment 分数，而是输出可组合的中间信号：

```json
{
  "event_importance": 0.75,
  "policy_relevance": 0.60,
  "policy_importance": 0.70,
  "llm_herding_signal": 0.50,
  "llm_institutional_signal": 0.65,
  "evidence": [
    "近期公告可能影响市场预期",
    "文本中出现机构调研和基金持仓相关信息"
  ]
}
```

## 10. risk_strategy：风险 / 策略标签

### 10.1 推荐方式总表

`risk_strategy` 模块主要由前面 6 个模块的标签组合生成。

| 标签 | 中文含义 | 推荐方式 | 是否需要 LLM / 模型 | 公式 / 计算方式 |
|---|---|---|---|---|
| `chase_high_risk` | 追高风险 | 规则为主 | 不需要 | `score = CHASE_HIGH_PRICE_RISE_WEIGHT * price_rise + CHASE_HIGH_NEAR_HIGH_WEIGHT * near_recent_high + CHASE_HIGH_VOLUME_EXPAND_WEIGHT * volume_expand + CHASE_HIGH_TURNOVER_WEIGHT * high_turnover + CHASE_HIGH_EMOTION_WEIGHT * short_term_emotion` |
| `false_breakout_risk` | 假突破风险 | 规则为主 | 不需要 | `score = FALSE_BREAKOUT_BREAKOUT_WEIGHT * breakout + FALSE_BREAKOUT_CLOSE_WEAK_WEIGHT * close_weak + FALSE_BREAKOUT_UPPER_SHADOW_WEIGHT * upper_shadow + FALSE_BREAKOUT_VOLUME_EXPAND_WEIGHT * volume_expand` |
| `liquidity_risk` | 流动性风险 | 规则为主 | 不需要 | `score = LIQUIDITY_LOW_TURNOVER_WEIGHT * low_turnover + LIQUIDITY_LOW_VOLUME_WEIGHT * low_volume + LIQUIDITY_WIDE_SPREAD_WEIGHT * wide_spread` |
| `drawdown_risk` | 回撤风险 | 规则为主 | 不需要 | `score = DRAWDOWN_NEAR_HIGH_WEIGHT * near_recent_high + DRAWDOWN_VOLATILITY_WEIGHT * volatility + DRAWDOWN_PRICE_RISE_WEIGHT * price_rise + DRAWDOWN_SUPPORT_DISTANCE_WEIGHT * support_distance` |
| `valuation_trap_risk` | 估值陷阱风险 | 混合 | 建议用 LLM | `score = low_valuation * fundamental_negative` |
| `overheated_risk` | 过热风险 | 规则为主 | 不需要 | `score = OVERHEATED_PRICE_RISE_WEIGHT * price_rise + OVERHEATED_VOLUME_EXPAND_WEIGHT * volume_expand + OVERHEATED_HIGH_TURNOVER_WEIGHT * high_turnover + OVERHEATED_EMOTION_WEIGHT * short_term_emotion` |
| `risk_control` | 风险控制 | 规则为主 | 不需要 | 推荐：`score = 1 - product(1 - r for r in [chase_high_risk, false_breakout_risk, liquidity_risk, drawdown_risk, valuation_trap_risk, overheated_risk])`；简化：`score = max(chase_high_risk, false_breakout_risk, liquidity_risk, drawdown_risk, valuation_trap_risk, overheated_risk)` |
| `position_control` | 仓位控制 | 规则为主 | 不需要 | `score = POSITION_CONTROL_RISK_WEIGHT * risk_control + POSITION_CONTROL_VOLATILITY_WEIGHT * volatility + POSITION_CONTROL_UNCERTAINTY_WEIGHT * uncertainty` |
| `wait_confirm` | 等待确认 | 混合 | 可用 LLM 辅助解释 | 推荐：`score = 1 - product(1 - s for s in [breakout * (1 - volume_expand), short_term_emotion * (1 - trend_confirmed), policy_driven * (1 - price_confirmed), sentiment_conflict])`；简化：`score = max(breakout * (1 - volume_expand), short_term_emotion * (1 - trend_confirmed), policy_driven * (1 - price_confirmed), sentiment_conflict)` |
| `observe_next_day` | 观察次日表现 | 规则为主 | 不需要 | `score = max(news_driven, policy_driven, market_attention_rise, volume_spike, large_intraday_move) * (1 - trend_confirmed)` |
| `avoid_emotional_trade` | 避免情绪交易 | 规则为主 | 不需要 | 推荐：`score = 1 - product(1 - s for s in [short_term_emotion, panic_selling, herding_effect, overheated_risk])`；简化：`score = max(short_term_emotion, panic_selling, herding_effect, overheated_risk)` |
| `take_profit_plan` | 止盈计划 | 规则为主 | 不需要 | `score = TAKE_PROFIT_PRICE_RISE_WEIGHT * price_rise + TAKE_PROFIT_NEAR_HIGH_WEIGHT * near_recent_high + TAKE_PROFIT_OVERHEATED_WEIGHT * overheated_risk + TAKE_PROFIT_DRAWDOWN_WEIGHT * drawdown_risk` |
| `stop_loss_plan` | 止损计划 | 规则为主 | 不需要 | `score = STOP_LOSS_BREAK_LOW_WEIGHT * break_recent_low + STOP_LOSS_DOWNTREND_WEIGHT * downtrend + STOP_LOSS_SENTIMENT_NEGATIVE_WEIGHT * sentiment_negative + STOP_LOSS_DRAWDOWN_WEIGHT * drawdown_risk` |

### 10.2 risk_strategy 关键变量

| 变量 | 含义 | 公式 / 来源 |
|---|---|---|
| `low_volume` | 低成交 | `clamp((1 - volume_ratio_5d) / VOLUME_SHRINK_RATIO_RANGE)` |
| `wide_spread` | 买卖价差大 | `clamp(spread_pct / spread_threshold)` |
| `low_valuation` | 低估值强度 | `max(low_pe, low_pb, high_dividend)` |
| `fundamental_negative` | 基本面负面强度 | LLM / 本地模型 / 财务规则输出 |
| `trend_confirmed` | 趋势确认度 | 例如 `MA5 > MA10 > MA20` 且 `current_price > MA5` 时较高 |
| `price_confirmed` | 价格确认度 | 突破后是否站稳关键位 |
| `sentiment_conflict` | 情绪冲突 | 可由规则或 LLM 判断，例如情绪偏正但价格下跌、消息利好但成交不跟随 |
| `uncertainty` | 不确定性 | `max(wait_confirm, sentiment_conflict, policy_uncertainty, news_uncertainty)` |

## 11. 哪些标签需要 LLM / 本地模型

### 11.1 强烈建议用 LLM 或本地模型

| 标签 | 原因 |
|---|---|
| `policy_driven` | 政策对不同资产、行业影响不同，不能只靠关键词 |
| `news_driven` | 需要理解事件重要性，以及行情是否可能由事件触发 |
| `institutional_behavior` | 机构行为常隐藏在研报、公告、基金持仓、调研信息里 |
| `valuation_trap_risk` | 低估值是否是陷阱，需要理解财报、公告、新闻中的基本面风险 |
| `fundamental_risk` | 需要识别盈利下滑、现金流恶化、资产质量问题、监管处罚等语义信息 |
| `herding_effect` | 文本里的从众、抱团、抢筹、净买入、净卖出等语义适合模型识别 |

### 11.2 建议用 FinBERT / 本地情绪模型

| 标签 / 基础变量 | 原因 |
|---|---|
| `sentiment_positive` | 基础正面情绪变量，供 `short_term_emotion` 等公式使用 |
| `sentiment_negative` | 基础负面情绪变量，供 `panic_selling`、`weak_sentiment` 等公式使用 |
| `sentiment_neutral` | 基础中性情绪变量，用于判断消息是否缺少方向性 |
| `short_term_emotion` | 需要文本情绪和短线行情共同判断 |
| `panic_selling` | 需要识别恐慌、抛售、负面情绪文本 |
| `weak_sentiment` | 需要识别负面情绪和市场冷淡 |

### 11.3 规则更稳定，不建议用 LLM

| 标签 |
|---|
| `asset` 下大部分资产类型标签 |
| `price` 下所有价格位置标签 |
| `volume` 下所有成交量 / 换手标签 |
| `trend` 下大部分趋势结构标签 |
| `low_pe`、`high_pe`、`low_pb`、`high_pb`、`high_dividend` |
| `market_attention_rise` |
| `sector_rotation` |
| `chase_high_risk` |
| `false_breakout_risk` |
| `liquidity_risk` |
| `drawdown_risk` |
| `overheated_risk` |
| `risk_control` |
| `position_control` |
| `observe_next_day` |
| `avoid_emotional_trade` |
| `take_profit_plan` |
| `stop_loss_plan` |

## 12. 精度优先的落地建议

如果目标是“数值准度优先”，推荐按下面优先级实现：

```text
第一优先级：历史分位数 percentile_rank
第二优先级：Robust Z-Score + Sigmoid
第三优先级：固定阈值 clamp
```

推荐实现方式：

```python
# 1. 有该标的历史数据时，优先用该标的历史分位
score = percentile_rank(current_value, asset_history_values)

# 2. 没有足够历史数据时，用行业分布
score = percentile_rank(current_value, industry_values)

# 3. 历史和行业数据都不足时，再退回固定阈值
score = clamp(current_value / threshold)
```

数据不足时：

```text
如果某个公式依赖的数据缺失，不要让 LLM 猜数值。
该标签可以不生成，或者生成 score = 0，并在 evidence 中记录数据缺失原因。
```

最终建议：

```text
规则负责数值稳定性；
FinBERT / 本地模型负责文本情绪；
LLM 负责政策、事件、机构行为、基本面风险等语义信号；
所有模型输出都尽量转成中间变量，再由公式合成最终标签分数。
```
