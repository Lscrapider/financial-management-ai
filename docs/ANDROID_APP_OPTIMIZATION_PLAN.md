# Android App 优化方案

## 当前优先级

Android 端优先服务报告、OCR、布控、研究和知识库材料流程。行情中心先隐藏入口，保留现有代码和接口对接，后续需要时再恢复。

## 页面原则

- 移除解释型二级说明卡，页面直接进入业务内容。
- 保持深色工作台体系，避免浅色筛选栏、浅色卡片和无意义装饰。
- 每个页面围绕核心任务组织，不复刻 Web 端的信息密度。
- 列表页只展示决策所需摘要，详细字段放入详情页或抽屉。
- 页面文案优先中文，按钮和状态说明必须直接表达动作或结果。

## 已确认处理

- 投资工作台保留今日行动、驾驶舱信号、观察池异动、资产分布和报告动态。
- 报告动态展示多条报告完成/处理中/失败记录，并显示完整日期时间。
- 顶部不显示管理员提示，不显示重复页面入口。
- 登录页独立存在，登录后默认进入投资工作台。
- 行情中心暂时隐藏：底部导航不展示“行情”，路由层拦截“行情中心”打开请求。

## 后续重点页面

- 观察风控：参考 Stitch「观察风控」屏，采用观察池与布控提醒两个覆盖内容；观察池保留选中标的卡片并集成成本/持仓/备注修改；布控提醒改为列表逐条点开，不展示选中提醒大卡，页面内提供添加布控提醒、阈值/启用/邮箱通知输入和输入回显。后端当前只提供涨跌幅阈值，不展示止盈/止损价配置。
- 报告研究：围绕报告标的、创建任务、轮询状态、历史版本和报告详情。
- 知识库材料与 OCR：围绕材料检索、OCR 任务、入库复核、片段证据和标签维护。
