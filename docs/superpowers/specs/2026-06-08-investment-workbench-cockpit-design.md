# 投资工作台驾驶舱设计规格

## 背景

当前投资工作台已经具备自选池异动、资产结构、风险布控和报告动态四类信息，但整体仍偏普通后台卡片页。第二轮 UI 提升的目标不是换皮，而是在现有 Element Plus 深色系统表格气质上，增加一个可记忆、可解释、可操作的“投资研究驾驶舱”焦点。

本规格采用 `$impeccable bolder / layout / delight / overdrive / animate` 的组合方向：

- `bolder`：强化投资工作台第一眼的主焦点，不做花哨装饰。
- `layout`：用“信号雷达 + 今日行动 + 三个任务区”的结构替代均质卡片感。
- `delight`：只在风险判断和操作反馈处加入细节。
- `overdrive`：把网页做出仪器反馈感，但不引入炫技型 WebGL、粒子或复杂 3D。
- `animate`：动效表达状态，不阻塞操作，并支持 reduced-motion。

## 设计原则

1. **沿用当前系统底色**
   页面底色、卡片底色、边框和圆角要匹配当前投资工作台和系统表格。以 `var(--el-bg-color)`、`var(--el-fill-color-lighter)`、`var(--el-border-color-light/lighter)` 为主，不新增一套偏蓝 cockpit 皮肤。

2. **语义色统一**
   继续使用现有金融语义色：
   - 主蓝：`#006be6`
   - 上涨/成功绿：`#57d188`
   - 预警黄：`#efbd48`
   - 风险红：`#dc4446`

3. **物理感来自交互逻辑**
   物理效果只表现为仪器反馈：雷达扫描、轻微磁吸、状态流动。禁止粒子背景、玻璃拟态、紫蓝渐变、持续闪烁、过度阴影。

4. **金融产品优先可信**
   页面应更有识别度，但仍像一个稳定可靠的投资研究工具。用户应该先看懂风险和下一步行动，而不是先注意到动效。

## 页面结构

### 顶部：保留当前页头

保留当前 `workbench-hero` 的标题、说明和操作按钮结构，只强化文案与间距，不把页头改成营销式 hero。

### 摘要区：保留 4 项摘要

保留当前四个摘要指标：

- 观察池标的
- 观察分组
- 上涨 / 下跌
- 报告异常

只做轻微视觉对齐和状态色一致化，不新增大型 hero metric 模板。

### 主体：信号驾驶舱

主体布局从均质三列卡片调整为：

1. **信号雷达主面板**
   作为页面第一视觉焦点，使用当前卡片底色和细边框。内部包含：
   - 雷达仪表：展示综合信号状态。
   - 信号列表：上涨标的、下跌标的、预警覆盖、报告生成。
   - hover 联动：鼠标经过信号行时，对应行轻微位移并高亮边框。

2. **今日行动侧栏**
   从现有数据推导出 2-3 个行动项，例如：
   - 复核下跌最大的观察标的
   - 补齐未覆盖的预警阈值
   - 进入报告工作台生成分析

   行动项使用轻微“磁吸”反馈：hover 时向上或向侧边移动 2-4px，边框使用主蓝或语义色增强。

3. **报告状态流**
   当存在运行中报告时，显示短线流动，表达“检索、分析、生成”。没有运行中报告时保持静态状态，避免持续动画疲劳。

4. **下方任务区**
   保留当前三个核心任务区：
   - 观察池异动
   - 资产视图
   - 布控与风险

   这些区域继续使用当前表格/卡片密度，只调整与雷达主面板的层级关系。

## 物理交互设计

### 雷达扫描

- 使用 CSS `conic-gradient` / `radial-gradient` 构建雷达盘。
- 扫描线慢速旋转，周期约 5-6 秒。
- 扫描只作为低强度背景反馈，不使用高亮闪烁。
- `prefers-reduced-motion: reduce` 时停止旋转，保留静态雷达盘。

### 信号行惯性

- 信号行 hover 时位移 4px 以内。
- 使用 `transform`，不动画 `left/top/width/height`。
- 动画时长 180-220ms，缓动使用 `cubic-bezier(.22, 1, .36, 1)` 或同等级 ease-out。
- 只联动当前行和雷达层，不引发全页面重排。

### 今日行动磁吸

- 行动项 hover 时轻微上浮或侧移。
- 边框从当前灰边框过渡到主蓝或语义色。
- 点击仍使用正常路由跳转，不新增中间确认。

### 报告状态流

- 只在存在 `generating_report`、`retrieving_knowledge`、`processing_current_scenes` 等运行态时出现。
- 完成、失败、空状态都不显示持续流动线。
- reduced-motion 下显示静态阶段条。

## 数据与状态逻辑

继续使用当前接口和 computed 数据，不新增后端接口：

- `listWatchGroups()`
- `listStockAlerts()`
- `listSceneReportTargets({ pageNum: 1, pageSize: 4 })`

新增或调整的前端派生状态：

- 综合信号状态：根据下跌数量、越界预警、报告异常、运行中报告推导为稳健、关注、风险。
- 今日行动列表：根据当前数据优先级生成，最多显示 3 项。
- 雷达分段：使用已有 `marketDirectionSummary`、`alertCoveragePercent`、`runningReportCount`、`failedReportCount`。

## 响应式规则

- 桌面：两列主体，左侧信号雷达，右侧今日行动和报告状态流。
- 平板：信号雷达仍在上方，今日行动移动到其下方，任务区两列。
- 手机：雷达盘缩小或横向压缩为信号条；任务区单列；所有按钮和行动项保持至少 44px 触控高度。

## 可访问性

- 雷达盘作为可视化摘要，不作为唯一信息来源。所有状态必须在文本信号列表中表达。
- 动效必须支持 `prefers-reduced-motion`。
- hover 效果必须有 focus-visible 等价反馈。
- 颜色不能作为唯一状态表达，风险/预警/运行中必须有文字标签。

## 性能约束

- 不引入 Three.js、WebGL、Canvas 或新动画库。
- 动画只使用 CSS transform、opacity、gradient background-position。
- 不在滚动事件中做 JS 动画。
- 不对大面积元素使用高成本 blur/filter。

## 验证标准

实现完成后至少验证：

1. `pnpm -C frontend-vue --pm-on-fail=ignore check:type`
2. `pnpm -C frontend-vue --pm-on-fail=ignore build:ele`
3. 目标文件 formatter、stylelint、ESLint 无 error。
4. 浏览器桌面视口检查：雷达、今日行动、任务区不重叠。
5. 390px 移动视口检查：无横向溢出，行动项可点击。
6. reduced-motion 模拟检查：雷达和状态流停止动画，但信息仍完整。

## 不做范围

- 不新增后端接口。
- 不新增单元测试，除非后续实现阶段发现有复杂派生逻辑需要回归保护。
- 不做整站主题重构。
- 不处理全量 lint 扫描生成目录的问题，该问题属于独立 polish 任务。
- 不替换现有 Element Plus 组件库。

已确认方向：

- 采用“信号驾驶舱”为主。

- 底色和当前系统表格保持一致。
